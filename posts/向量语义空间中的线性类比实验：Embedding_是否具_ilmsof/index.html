向量语义空间中的线性类比实验：Embedding 是否具备代数结构？

1. 问题动机

在词向量与语义表示研究中，一个经典问题是：

向量空间是否具备“线性语义结构”？例如： A − B + C 是否能够得到语义上的 D？

早期的 word2vec 曾在实验中展示出类似现象，例如：

king − man + woman ≈ queen

这一结果引发了“语义可线性表达”的假设。

而在现代 embedding 模型中，例如 OpenAI text-embedding-3-large，这种结构是否仍然成立，是一个值得验证的问题。

2. 实验设计

本实验的目标是：

使用 embedding 向量进行线性组合，并通过余弦相似度检索最近语义结果，观察其是否具备稳定类比能力。

我们使用如下计算形式：

v = emb(A) - emb(B) + emb(C)

随后在语义空间中计算：

cos(v, emb(x))

并取最相近的语义文本作为结果。

3. 实验代码（核心实现）

以下代码基于 OpenAI embedding API 实现：

from openai import OpenAI
import numpy as np

client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

def emb(text):
    return np.array(
        client.embeddings.create(
            model="text-embedding-3-large",
            input=text
        ).data[0].embedding
    )

def cosine(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# =========================
# 实验1：北京 - 中国 + 法国
# =========================

A = "北京"
B = "中国"
C = "法国"

v = emb(A) - emb(B) + emb(C)

candidates = [
    "巴黎", "柏林", "伦敦", "东京", "纽约", "莫斯科", "北京"
]

scores = []
for c in candidates:
    scores.append((c, cosine(v, emb(c))))

scores.sort(key=lambda x: x[1], reverse=True)

print("Top results:")
for s in scores[:5]:
    print(s)

可以观察到：

  • 所有结果集中在“城市语义簇”
  • 巴黎通常接近顶部，但不稳定第一
  • 向量差异较小（0.5~0.6 区间）

5. 结果分析

该结果说明：

  1. embedding 空间确实具有语义聚类结构 城市类概念在空间中形成局部连续区域

  2. 但不存在稳定的线性变换结构 即：

    A − B + C ≠ 稳定等于 D

  3. cosine similarity 在此空间中表示的是：

    向量方向一致性（语义相关性），而非逻辑关系

6. 机制解释

现代 embedding（如 OpenAI embedding）采用对比学习目标，其优化方向是：

  • semantic similarity
  • retrieval ranking
  • multilingual alignment

而不是：

explicit linear algebra structure

因此其空间更接近：

非线性语义流形（manifold）

而非：

可严格线性组合的几何空间

7. word2vec 的实验代码和结果

import numpy as np
import gensim.downloader as api

# =========================
# 1. 加载 word2vec
# =========================
model = api.load("word2vec-google-news-300")

print("cache dir:", api.BASE_DIR)

# =========================
# 2. 安全 embedding（避免 KeyError）
# =========================
def emb(text):
    """
    word2vec 是严格词表模型
    不存在的词直接返回 None
    """
    if text in model:
        return model[text]
    else:
        return None

# =========================
# 3. cosine 相似度
# =========================
def cosine(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# =========================
# 4. 安全评分函数
# =========================
def safe_score(v, word):
    vec = emb(word)
    if vec is None or v is None:
        return -1e9
    return cosine(v, vec)

# =========================
# 实验1：beijing - china + france
# =========================

A, B, C = "beijing", "china", "france"

v = emb(A) - emb(B) + emb(C)

candidates = [
    "paris", "berlin", "london", "tokyo",
    "new york",   # ⚠ 注意：必须是空格形式
    "moscow", "beijing"
]

scores = []
for c in candidates:
    scores.append((c, safe_score(v, c)))

scores.sort(key=lambda x: x[1], reverse=True)

print("\n=== Analogy: beijing - china + france ===")
for s in scores[:5]:
    print(s)

# =========================
# 实验2：king - man + woman
# =========================

A, B, C = "king", "man", "woman"

v = emb(A) - emb(B) + emb(C)

candidates = ["queen", "king", "woman", "man", "prince", "princess"]

scores = []
for c in candidates:
    scores.append((c, safe_score(v, c)))

scores.sort(key=lambda x: x[1], reverse=True)

print("\n=== Analogy: king - man + woman ===")
for s in scores[:5]:
    print(s)

8. 对比结论（与 word2vec）

word2vec 在早期实验中更容易出现线性类比现象，其原因是：

  • 基于共现矩阵分解
  • 空间近似低秩线性结构

但其缺点是：

  • 泛化能力较弱
  • 语义表达能力有限

现代 embedding 则：

  • 放弃严格线性结构
  • 换取更强语义表示能力与检索性能

9. 总结

本实验表明：

向量空间中的“语义接近”可以通过 cosine similarity 稳定度量，但“向量加减等于语义推理”并不构成稳定规律。

换句话说：

  • embedding 是语义空间
  • cosine 是相似度函数
  • 但不是代数推理系统
