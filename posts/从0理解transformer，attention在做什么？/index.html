从 0 理解 Transformer：Attention 到底在干什么？

面向有编程基础（Python）的读者，从直觉 + 代码 + 数学三层彻底讲清楚 Transformer 核心机制。

一、为什么需要 Transformer？

在 Transformer 出现之前，主流方法是：

  • RNN / LSTM（顺序计算，无法并行）
  • CNN（局部建模能力强，但难以捕捉长距离依赖）

它们的问题：

  • ❌ 无法高效并行（RNN）
  • ❌ 长距离依赖难以建模

👉 Transformer 的核心思想：

用 Attention 机制，让每个 token 直接“看到”所有 token

二、Attention 的核心公式

Transformer 的核心公式如下：

Attention(Q,K,V)=softmax(QKTdk)V\text{Attention}(Q,K,V)=\text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)VAttention(Q,K,V)=softmax(dk​​QKT​)V

三、从 for 循环理解 Attention

先不用矩阵，直接用代码理解：

for i in range(n):
    for j in range(n):
        score[i][j] = dot(q[i], k[j])

👉 含义：

第 i 个 token，计算它与所有 token 的相关性

接着做 softmax：

softmax = 归一化权重，是对一整行进行处理，而不是单个元素

for i in range(n):
    score[i] = softmax(score[i])

最后融合信息：

for i in range(n):
    output[i] = 0
    for j in range(n):
        output[i] += score[i][j] * v[j]

四、矩阵形式（for 循环的并行化）

上述代码可以压缩为：

scores = Q @ K.T
A = softmax(scores)
output = A @ V

👉 本质：

  • QK^T = 所有 token 两两相似度
  • softmax = 归一化权重
  • * V = 信息加权融合

五、Embedding 是什么？

1. 本质

Embedding = 一个可训练的查表

Embedding = [
    [0.1, 0.3, 0.2, 0.7],  # token1
    [0.9, 0.5, 0.1, 0.2],  # token2
]
x = Embedding[token_id]

2. 含义

  • 每个 token → 一个向量
  • 向量空间 = 语义空间

六、Q / K / V 是什么？

1. 来源

Q = X @ Wq
K = X @ Wk
V = X @ Wv

Wq / Wk / Wv 是三个不同的“特征提取器”，都是可学习的线性变换

向量直觉含义
embedding（X）这个词是什么
Q（X @ Wq）这个词想找什么
K（X @ Wk）这个词能被怎么找
V（X @ Wv）这个词能提供什么信息

✔️ 因为“一个词不止一种角色”

示例

句子：

“I gave a book to her”

对于 her，在 Attention 中可以承担不同角色：

  • 作为 Q：我要找谁给我东西？
  • 作为 K：我是“接受者”，可以被匹配到
  • 作为 V：我携带“接受者”的语义信息

同一个词，不同用途如下：

角色含义
Q主动查询
K被匹配
V提供信息

如果只用一个 embedding：

❌ 无法区分这些角色

2. 本质与语义

Q / K / V 是 embedding 的三种不同投影：分别负责“查询”“匹配”“传递信息”。

向量含义
Q我想找什么
K我能被怎么匹配
V我提供什么信息

3. 为什么要三套？

如果：

Q = K = V = X

👉 只能做简单相似度计算，表达能力不足

👉 使用三套参数：

  • 解耦“匹配”和“信息”
  • 提升模型表达能力

七、矩阵乘法本质（关键理解）

矩阵乘法：

C=A×BC = A \times BC=A×B

其中：

C[i][j]=A[i]⋅B[j]C[i][j] = A[i] \cdot B[j]C[i][j]=A[i]⋅B[j]

👉 即：

一行 × 一列 = 点积

八、完整 Attention 流程（带例子）

假设：

Q = [[1, 2], [3, 4]]
K = [[5, 6], [7, 8]]
V = [[10, 10], [20, 20]]

Step 1：计算 QK^T

scores = [
    [17, 23],
    [39, 53],
]

Step 2：softmax

A = [
    [0.0025, 0.9975],
    [0, 1],
]

Step 3：乘 V

output = [
    [19.975, 19.975],
    [20, 20],
]

👉 含义：

每个 token = 所有 token 信息的加权组合

九、Attention 的本质

一句话总结：

每个 token 都在问：“我应该关注谁？”

过程：

  1. 用 Q 查询
  2. 用 K 匹配
  3. 用 V 取信息

十、复杂度分析

设：

  • n = token 数
  • d = 向量维度

时间复杂度

O(n2⋅d)O(n^2 \cdot d)O(n2⋅d)

空间复杂度

O(n2)O(n^2)O(n2)

👉 问题：

  • 长文本会爆炸（n²）

十一、为什么 Transformer 很强？

1. 全局建模

任意 token 之间直接连接

2. 并行计算

矩阵乘法 → GPU 友好

3. 表达能力强

Q / K / V 解耦信息流

十二、程序员视角总结

# 输入
# X: n × d

# 投影
Q = X @ Wq
K = X @ Wk
V = X @ Wv

# Attention
scores = Q @ K.T
A = softmax(scores)
output = A @ V

十三、一句话终极总结

👉 Transformer = 一个全连接信息传播网络

👉 Attention = 基于相似度的信息加权聚合
