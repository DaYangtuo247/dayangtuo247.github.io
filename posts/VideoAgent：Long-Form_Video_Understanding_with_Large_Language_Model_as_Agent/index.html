VideoAgent：Long-Form Video Understanding with Large Language Model as Agent

人工智能paper # 深度学习# NLP# CLIP# GPT
May 11, 2026
15 min read
—

原文：https://arxiv.org/pdf/2403.10517.pdf

  • 题目：VideoAgent: Long-Form Video Understanding with Large Language Model as Agent（视频智能体：基于大语言模型作为代理的长视频理解）
  • 会议期刊名称：ECCV 2024 (欧洲计算机视觉国际会议，CV顶会)
  • 年份：2024
  • 作者：Xiaohan Wang, Yuhui Zhang, Orr Zohar, and Serena Yeung-Levy
  • 单位：Stanford University (斯坦福大学)

问题

  • 研究背景：长视频理解（从几分钟到几个小时不等）是计算机视觉领域的一个巨大挑战。
  • 核心问题定义：如何在不超出大模型上下文长度限制、不产生庞大计算开销的前提下，从超长、多模态的视频序列中推理并提取有效信息以准确回答给定的问题？
  • 现存痛点：人类看长视频找答案时，并不是逐帧看完，而是先“快进扫一眼”，然后“针对问题找关键帧”，最后总结答案。但目前的AI模型大多缺乏这种交互式的推理与规划能力。

前人工作进行分类陈述

在VideoAgent之前，学术界主要通过以下两类方法来处理长视频：

  • 第一类：压缩稀疏化方法（Compressive sparsity methods）

    • 核心思想：试图将长视频通过降维的手段，压缩成尽可能小的嵌入向量（Embeddings）、时空图谱或者纯文本描述，然后再喂给大模型。
    • 代表性方法：MovieChat（合并相似帧特征），Chat-UniVi（聚类压缩），LLoVi（把视频全转成字幕然后让LLM读）。
    • 缺点：暴力压缩不可避免地会导致细粒度关键信息的丢失。更为致命的是，当面对长达几小时的视频时，即使压缩成了文本，文本量依然巨大，大语言模型（LLM）会陷入“Lost in the middle（迷失在中间）”的困境，容易被冗余噪音带偏，且计算效率极其低下。

  • 第二类：选择性压缩方法（Selective-compressive methodologies）

    • 核心思想：利用用户输入的问题（Query）作为向导，对视频进行子采样，只抽取和问题相关的视频帧。
    • 代表性方法：R-VLM，SeViLA。
    • 缺点：这些方法大多是**“单轮静态检索”**（Single iteration）。它们通常直接拿原始问题去检索，如果问题涉及到复杂的时序因果（例如：“在这名男子离开房间后，桌子上留下了什么？”），单次检索很容易找错时间点。它们缺乏人类那种“看了前半段，发现线索不够，再换个关键词去找后半段”的多轮交互推理能力。

本文提出的改进思路、创新点和贡献

  • 改进思路：受人类认知的启发，本文放弃了“让模型直接处理长视频”的死胡同，而是提出一个代理系统（VideoAgent），让LLM作为大脑，视觉语言模型（VLM）作为“翻译”，CLIP作为“眼睛”，通过多轮迭代的方式，动态搜索信息直至足以回答问题。

  • 创新点与贡献：

    1. 首次将长视频理解定义为带有状态更新和反思机制的序贯决策（MDP）过程。
    2. 提出了动态片段检索和自我评估机制。
    3. 实现了极高的“帧效率”，在多个权威数据集上取得SOTA（当前最佳），但平均只需抽取不到9帧。

  • 对前人方法的改进对比（表格形式）

P2 前人方法的缺陷VideoAgent 作出的改进 (本文创新)
暴力压缩/均匀采样：导致长上下文信息冗余，大量算力浪费在无关帧上。高帧效率的按需采样：大模型主导动态抽帧，只需平均8.4帧即可击败需要180帧的前人方法，彻底解决上下文爆炸。
单次/静态检索：直接用原始问题去搜，难以应对需要跨时序组合线索的复杂问题。多轮迭代与分段检索：LLM可以根据当前状态重写检索Query，并限制在特定视频片段（Segment）内查找，增强时序推理。
缺乏信息量感知：不知检索到的线索是否足以作答。自我反思机制（Self-reflection）：引入三阶段思维链，Agent自主打分（1至3分），当认为线索充分时主动停止检索。

算法建模

本文将长视频的理解过程公式化为一个序列集：{(st,at,ot)∣1≤t≤T}\{(s_t, a_t, o_t) | 1 \leq t \leq T\}{(st​,at​,ot​)∣1≤t≤T}，其中 sss 是状态， aaa 是动作，ooo 是观察。

公式 2：行动决策与自我反思 (核心推理引擎) y^=PredictAnswer(F∗l,st,q)\hat{y} = \text{PredictAnswer}(F*l, s_t, q)y^​=PredictAnswer(F∗l,st​,q) c=SelfReflect(Fl,st,q,y^)c = \text{SelfReflect}(F_l, s_t, q, \hat{y})c=SelfReflect(Fl​,st​,q,y^​) at=I∗[c≥C]a_t = \mathbb{I}*{[c \geq C]}at​=I∗[c≥C] 讲解：FlF_lFl​ 是作为核心引擎的大语言模型（LLM），qqq 是原问题。LLM 首先基于当前状态 sts_tst​ 试着回答问题得到 y^\hat{y}y^​，然后它需要自我打分（置信度 ccc 从1到3）。如果置信度 ccc 大于等于我们设定的阈值 CCC（比如充分自信），动作 ata_tat​ 就是“输出答案并终止”；否则就是“继续寻找信息”。

公式 3：动态观察与状态更新 (填补信息差) h=FindMissingInfo(F∗l,st,q)h = \text{FindMissingInfo}(F*l, s_t, q)h=FindMissingInfo(F∗l,st​,q) ot=RetrieveFrames(Fc,v,h)o_t = \text{RetrieveFrames}(F_c, v, h)ot​=RetrieveFrames(Fc​,v,h) s∗t+1=Merge(s∗t,F∗v(o∗t))s*{t+1} = \text{Merge}\Big(s*t, F*{v}(o*t)\Big)s∗t+1=Merge(s∗t,F∗v(o∗t)) 讲解：如果信息不够，LLM 会推导出缺失了什么信息 hhh（包含新的关键词和查找片段）。然后调用多模态检索工具 FcF_cFc​（如CLIP）在视频 vvv 中提取匹配的图片 oto_tot​。最后用 VLM FvF_vFv​ 翻译成文本，合并到旧状态中，形成新的状态 s∗t+1s*{t+1}s∗t+1，进入下一轮循环。

技术框图

讲解：

  1. 最左侧是输入的超长视频和具体问题（如：“一开始狗在玩什么？”）。
  2. 图的下方是一个闭环的 Agent 系统。LLM（大语言模型，即中央大脑） 处于中心控制地位；VLM（视觉语言模型，负责看图说话） 和 CLIP（图文匹配模型，负责在海量视频里精准搜图） 作为被大脑调用的外部工具。
  3. 我们可以清晰地看到红色的决策流：Generate Captions（看大致情况） -> Predict & Self Reflect（尝试回答并评估信心） -> Find Missing Information（如果信心不足，决定需要再看什么） -> Retrieve Frames（通过CLIP抓取新图） -> 循环回左侧。
  4. 这个框架极其优雅地将“处理海量视觉像素”这一重计算任务，转换成了“大模型文本逻辑推理+轻量级多模态检索”的过程。

框图中的步骤拆解

讲解：整个系统分为四个严密的执行步骤：

  1. 获取初始概况 (Initial State)： 先在长视频中均匀抽取几帧（比如第50, 250, 450帧），用 VLM 翻译成文字：“50s: 狗在屋里走。250s: 狗在洗澡…”。这一步给了大模型一个“视频主线大纲”。

  2. 行动预测与自我反思 (Predict & Self Reflect)： 对于问题“一开始狗在玩什么”，大模型一看这大纲，没有“玩什么”的信息。强制让大模型推理后，自我反思模块会给出 Confidence: Low。系统判定：需要寻找新信息。

  3. 搜集新线索 (Find Missing Info)： 此时大模型展现出惊人的规划能力，它知道要回答“一开始”，所以把检索范围（Segment）缩小到 “1-50帧”，并且生成检索关键词：“a dog is playing with something in a room”。调用 CLIP 工具，瞬间在视频前50帧里找出了第53帧的最匹配画面。

  4. 状态更新 (Update State)： 拿到第53帧后，再次调用 VLM，将其转化为文字：“狗在玩一块冰（ice）”。这段文字被添加到上下文记忆中。系统进入第二轮，此时大模型发现信息充分，自信满分，立刻输出正确选项“A. ice”，流程结束。

实验数据、指标和实现细节

  • 数据集：
    • EgoSchema：第一人称视角的长视频基准测试。包含5000个多项选择题，每个视频长达3分钟（对传统模型极度困难）。
    • NExT-QA：日常交互视频，平均长度44秒，重点考察模型的**时序（Temporal）和因果（Causal）**推理能力。
  • 评价指标：选择题的准确率（Accuracy）。
  • 模型实现细节：
    • 检索器工具（CLIP）：选用 EVA-CLIP-8B-plus。
    • 字幕生成工具（VLM）：EgoSchema选用LaViLa，NExT-QA选用CogAgent。
    • 核心大脑（LLM）：采用 OpenAI 的 GPT-4 (gpt-4-1106-preview版本)，最大迭代限制设为 3 轮。
    • 视频解码帧率：极低的 1 fps（每秒仅1帧）。

SOTA 对比

讲解：跟领域内其它主流SOTA（State-of-the-Art）方法的比较。

  1. EgoSchema 表现 (Table 1)： 我们的 VideoAgent 在完整数据集上取得了 54.1% 的准确率，以高达 3.8% 的优势击败了并发的强力方法 LLoVi。更令人惊讶的是效率：VideoAgent 平均每个视频只看了 8.4 帧！而 LLoVi 暴力生成了 180 帧的字幕，计算成本高昂却效果更差。
  2. 匹敌闭源大厂模型 (Table 2参考)： 仅用这8.4帧的轻量架构，性能逼近了直接输入海量画面的谷歌闭源多模态巨头模型 Gemini 1.0 Pro (55.7%)。
  3. 复杂逻辑理解 (Table 3 NExT-QA)： 在NExT-QA中，VideoAgent 达到 71.3% 的零样本准确率，超越了所有之前的无监督/零样本方法。特别是在最难的 ATP-hard 子集（专门考察长时序推理）上，准确率高达 58.4%，验证了分段检索和多轮反思机制对于处理因果时间序列数据的巨大优越性。

超参数动态性能与消融实验

讲解：通过内部自比（消融实验）揭示其动力学特性。

  1. 看多少帧合适？（帧效率图 Fig 3左） 如图3左侧蓝线和红线对比，VideoAgent(红线)仅用不到10帧就突破了60%（在EgoSchema 500题子集上），而均匀采样基线(蓝线)哪怕增加到180帧，准确率也卡在59.6%无法进一步提升。这证明了：冗余的“废话信息”会分散大模型注意力，看准少数的“关键帧”才是长视频理解的核心。
  2. 因题制宜的动态消耗（Fig 3右） 模型拥有类似人的感知：面对最简单的“描述类”问题，模型平均两轮就能答对（平均消耗5.9帧）；而面对复杂的“因果”和“时序”推理问题，模型会自动选择多搜索几轮（平均消耗7.1帧和7.8帧）。
  3. 核心机制的消融（Table 5）
    • 如果我们去掉**“Self-Evaluation（自我评估阈值）”**，强制模型每道题都搜满3轮，提取的平均帧数从8.4飙升到11.8，但准确率反而从60.2%下降到59.6%。这证明了“知止而后定”的重要性。
    • 如果去掉**“Segment Selection（片段选择）”**，任由全局乱搜，准确率大跌 3.6%。时序隔离是建立正确因果逻辑的基础。

CASE

总结

  • 学术贡献：VideoAgent 颠覆了过去“把长视频直接倒给大模型”的特征级暴力堆砌路线，开创性地利用LLM的推理能力来反向主导视觉信息的抓取。
  • 方法优势：做到了极致的“高精尖”——准确率 SOTA，平均仅需 8.4 帧，极大地降低了长上下文情况下的计算负担，并且有效避免了“Lost in the middle（迷失在长文中）”的问题。
