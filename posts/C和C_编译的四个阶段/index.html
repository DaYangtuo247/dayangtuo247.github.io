C++ 编译全过程与构建工具（g++ / make / cmake）

我们通常使用 g++ main.cpp -o app 来编译 C++ 程序。这条看似简单的命令，背后实际上隐藏了完整的编译流水线。

一、 C++ 编译的四个核心阶段

完整的编译流程如下：

源文件 ──(预处理)──> 预处理文件 ──(编译)──> 汇编文件 ──(汇编)──> 目标文件 ──(链接)──> 可执行文件
 .cpp    [-E]       .i / .ii    [-S]       .s     [-c]       .o               a.out / .exe

1️⃣ 预处理（Preprocessing）

  • 命令：g++ -E main.cpp -o main.ii
  • 作用：纯文本替换。展开 #include 头文件、替换宏（#define）、处理条件编译（#ifdef）。

2️⃣ 编译（Compilation）

  • 命令：g++ -S main.ii -o main.s
  • 作用：进行语法和语义检查，将代码翻译成人类可读的汇编代码。

3️⃣ 汇编（Assembly）

  • 命令：g++ -c main.s -o main.o
  • 作用：将汇编代码转换为机器能看懂的机器码，生成目标文件（.o）。此时包含符号表，但可能还有未解析的符号（如外部函数）。

4️⃣ 链接（Linking）

  • 命令：g++ main.o a.o -o app
  • 作用：符号解析与地址重定位。将多个 .o 文件以及依赖的库文件打包合并，生成最终的可执行文件。
  • (注：如果不指定 -o 名字，Linux/Mac 下默认生成 a.out)

小结

也可以使用一次性的编译命令：

g++ main.cpp a.cpp b.cpp -o app
-o可以不加，默认生成a.out（assembly output）

编译参数的含义：

参数作用
-E只进行预处理，输出预处理结果
-S只进行编译，输出汇编代码
-c只进行汇编，输出目标文件（.o）
-o指定输出文件名

二、 编译相关的核心概念

  • 独立编译：编译器一次只处理一个 .cpp 文件，不会跨文件查找定义。
  • 头文件的本质：.h 文件只是被 #include 原样复制到源文件中，不参与独立编译。
  • 常见链接错误：
    • undefined reference：声明了函数但找不到定义。
    • multiple definition：同一个变量或函数被重复定义。
  • extern 与 static：
    • extern：告诉编译器“这个变量/函数在其他文件中定义过了，去别处找”。
    • static：将变量/函数的作用域限制在当前文件（或函数）内，防止链接时命名冲突。

三、 g++、make 与 cmake 的进化史

当项目只有一两个文件时，直接用 g++ 没问题。但当项目变成成百上千个文件时，问题就来了。

1. make：解决“重复编译”问题

如果每次都用 g++ *.cpp，所有文件都会重新编译，极度耗时。

  • 作用：自动化构建工具。它通过读取 Makefile 中的规则，对比文件修改时间，只编译修改过的文件，极大提高效率。

2. cmake：解决“Makefile 手写太难”问题

大型跨平台项目的 Makefile 手写起来极其复杂且难以维护。

  • 作用：生成 Makefile 的工具。开发者只需编写简单的 CMakeLists.txt，cmake 就能自动为不同平台生成对应的 Makefile。

3. 三者的协作流程

在现代 C++ 项目开发中，它们的标准协作流程如下：

 CMakeLists.txt
       │
       ▼ (cmake 解析规则)
    Makefile
       │
       ▼ (make 执行调度)
 调用 g++ 进行增量编译
       │
       ▼
  生成可执行文件

一句话总结

👉 g++ 负责干活，make 负责调度，cmake 负责制定规则。

一个简单的示例

//main.cpp
#include <iostream>
#include "header.h"

int main() {
    std::cout << "CONST_VALUE = " << CONST_VALUE << std::endl;

    funcA();  // 调用 a.cpp 的函数
    funcB();  // 调用 b.cpp 的函数
    funcA();  // 再调用 a.cpp 的函数

    std::cout << "globalVar in main = " << globalVar << std::endl;
    return 0;
}

// a.cpp
#include <iostream>
#include "header.h"

// 定义全局变量（唯一一份）
int globalVar = 10;

// 文件内部静态变量，只在 a.cpp 可见
static int fileLocalA = 0;

void funcA() {
    fileLocalA++;
    globalVar++;
    std::cout << "[funcA] fileLocalA = " << fileLocalA
              << ", globalVar = " << globalVar << std::endl;
}

// b.cpp
#include <iostream>
#include "header.h"

// 文件内部静态变量，只在 b.cpp 可见
static int fileLocalB = 100;

void funcB() {
    fileLocalB += 5;
    globalVar += 5;
    std::cout << "[funcB] fileLocalB = " << fileLocalB
              << ", globalVar = " << globalVar << std::endl;
}

// header.h
#pragma once

// 函数声明（接口）
void funcA();
void funcB();

// 全局变量声明
extern int globalVar;

// 常量
constexpr int CONST_VALUE = 100;

一个示例Makefile文件

app: main.o a.o b.o
	g++ main.o a.o b.o -o app

main.o: main.cpp
	g++ -c main.cpp

a.o: a.cpp
	g++ -c a.cpp

b.o: b.cpp
	g++ -c b.cpp
