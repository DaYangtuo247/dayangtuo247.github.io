从 Transformer 到 CLIP：一次对多模态 AI 的整体理解

参考资料：

  • VIT （Vision Transformer）深度讲解
  • 多模态模型CLIP深度讲解
  • 【硬核】一个视频讲透视觉语言模型 | 从像素到Token的完整旅程

为什么会写这篇

最开始学习 AI 的时候，会感觉整个领域非常割裂：

  • NLP 在讲 Transformer
  • 图像在讲 CNN
  • 音频在讲 Whisper
  • 视频又是一套自己的东西

后来真正开始理解 ViT、CLIP、多模态之后，会突然发现：其实它们底层正在逐渐统一。

这篇笔记就是一次从：

Transformer
→ ViT
→ CLIP
→ 多模态 LLM

的完整理解过程。

一、Transformer 到底强在哪里

原始论文：《Attention Is All You Need》

Transformer 最革命的地方，不只是性能。

而是：它第一次把“关系建模”抽象成了统一结构。

以前：

领域主流模型
NLPRNN / LSTM
图像CNN
视频3D CNN
音频RNN

每个领域都有自己的模型。

Transformer 出现后：

所有东西开始：

token化
→ embedding
→ attention

统一了。

二、Attention 的本质

核心公式：

Attention(Q,K,V)=softmax(QKTdk)V\mathrm{Attention}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)VAttention(Q,K,V)=softmax(dk​​QKT​)V

真正重要的不是公式本身。而是：token 与 token 之间建立动态关系。

以前：

  • CNN 只能局部看
  • RNN 只能按顺序传递

Transformer：

任意 token 都能直接通信

例如：

第1个 token
直接关注
第1000个 token

三、ViT：Transformer 如何处理图片

论文：AN IMAGE IS WORTH 16X16 WORDS: TRANSFORMERS FOR IMAGE RECOGNITION AT SCALE

ViT 的核心思想非常简单：把图片切成 patch，当成单词。

例如：

224×224 图片
↓
16×16 patch
↓
14×14 = 196 个 patch

于是：

图片 = 196 个 token

1. patch embedding

一个 patch：

16×16×3
= 768

先 flatten：

[pixel values...]

再通过 Linear Projection：

768 → embedding_dim

这一步本质上类似：

word embedding

只是：

  • NLP 是单词 embedding
  • ViT 是 patch embedding

2. ViT 真正做了什么

不是：

“识别图片”

而是：

把二维图像转换成 Transformer 能处理的一维序列。

四、为什么后来又加入 CNN

原始 ViT 有个问题：

对图像结构几乎没有先验。

CNN 天然适合：

  • 边缘
  • 局部纹理
  • 平移不变性

所以后来很多模型开始：

CNN + Transformer

例如：

  • Hybrid ViT
  • Swin Transformer
  • ConvNeXt

五、CLIP 到底是什么

论文：Learning Transferable Visual Models From Natural Language Supervision

很多人第一次接触 CLIP，会以为：

“这是个图片文本匹配模型。”

其实不完全对。

CLIP 真正做的是：

学习一个统一的图文语义空间。

六、CLIP 的结构

CLIP 本质：

Image Encoder + Text Encoder

通常：

部分模型
Image EncoderViT
Text EncoderTransformer

图片路径

图片
↓
ViT
↓
image embedding

文本路径

文本
↓
Transformer
↓
text embedding

七、CLIP 如何训练

CLIP 使用：对比学习（Contrastive Learning）

核心思想：

正样本拉近
负样本推远

例如：

图片文本
猫”a cat” ✅
猫”a dog” ❌

相似度计算

通常使用余弦相似度：

cos⁡(θ)=A⋅B∣A∣∣B∣\cos(\theta)=\frac{A\cdot B}{|A||B|}cos(θ)=∣A∣∣B∣A⋅B​

但这里需要注意：余弦相似度不是 CLIP 的核心。

真正核心是：

embedding space

八、CLIP 为什么不需要人工 label

这是 CLIP 最厉害的地方。

以前：

图片
→ 人工标注

CLIP：

互联网图片 + 文本

直接训练。

例如：

图片：猫
配文："my lovely cat"

虽然不严格。

但：

数据量足够大时，统计规律会自动形成语义结构。

九、False Negative 问题

对比学习有个经典问题：

"a cat"
"a kitten"

其实语义接近。

但训练中可能互相作为负样本。

CLIP 并不能完全避免。

它依赖的是：

海量数据
+
统计规律

而不是：

严格监督

十、CLIP 最终得到的到底是什么

不是：

一个匹配分数

而是：

统一语义 embedding

例如：

猫图片 embedding
≈
“cat” embedding

最终：

cat
kitten
tiger
lion

会自然聚类。

十一、多模态 LLM 是怎么来的

这里开始进入 GPT-4V 一类模型的核心。

LLM 本身：

只能处理 token

它并不懂图片。

所以：

需要先把图片转换成：

visual tokens

十二、视觉 token 是怎么进入 LLM 的

流程：

图片
↓
ViT
↓
visual tokens
↓
MLP / Projection
↓
映射到 LLM embedding space

这里：

MLP 的作用只是：

对齐维度和分布。

真正建立：

图片 ↔ 文本

联系的是：Attention

十三、文本和图片是如何联系起来的

这是理解多模态最关键的一步。

输入实际上会变成：

[V1][V2][V3]...

[T1][T2][T3]...

其中：

  • V 是 visual tokens
  • T 是 text tokens

Transformer 会：

所有 token 两两 attention

于是：

“猫”

这个文本 token。

会自动关注：

猫耳朵
猫尾巴

对应视觉 token。

十四、现代多模态模型的本质

最终：

图片 → visual tokens
音频 → audio tokens
视频 → video tokens
文本 → text tokens

全部统一成：

embedding sequence

然后：

Transformer

统一处理。

这也是：

“万物皆 token”

这句话的来源。

十五、Transformer 为什么又让人头疼

虽然统一了。

但：太耗性能了。

Attention 复杂度：O(N2)O(N^2)O(N2)

token 数一大：显存和计算直接爆炸。视频尤其恐怖。

例如：

16 帧
×
196 patch
=
3136 tokens

Attention：

3136²
≈ 1000万关系

十六、为什么移动端很难

Transformer：

  • 显存大
  • 功耗高
  • KV Cache 巨大
  • Attention 太重

所以：

很多边缘设备现在仍然大量使用 CNN。

十七、现在行业正在做什么

最近几年开始重新研究：

  • Linear Attention
  • Mamba
  • RWKV
  • MoE
  • Hybrid 架构

核心目标：降低复杂度。因为Transformer 虽然强。但太贵了。

十八、现在最大的感受

学到后面会发现 Transformer 已经不像传统算法，更像通用信息处理架构

它真正统一的不是：

  • NLP
  • 图像
  • 视频

而是：信息表示 本身。

十九、最后的大统一

现代 AI 越来越像：

任何输入
↓
tokenization
↓
embedding
↓
Transformer
↓
关系建模
↓
生成/推理

于是：

  • GPT
  • CLIP
  • Whisper
  • VideoMAE
  • GPT-4V

这些模型本质上都开始变成：“不同模态 token 的统一处理系统”。
