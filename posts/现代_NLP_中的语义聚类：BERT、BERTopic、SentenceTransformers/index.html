现代 NLP 中的语义聚类：BERT、BERTopic、SentenceTransformers

很多人第一次看到 BERTopic 时，会误以为它是某种 “BERT 升级版模型”。

实际上两者完全不是一个层级的东西：

名称本质
BERT语言模型（Language Model）
BERTopic主题建模框架（Topic Modeling Pipeline）

一句话理解：

BERT 负责理解语义
BERTopic 负责利用语义做聚类

BERT 到底在做什么？

BERT 最核心的能力，其实是：

文本 -> embedding 向量

例如：

“苹果发布新手机”
“iPhone新品上市”

虽然词完全不同，但在 embedding 空间中会非常接近。

因为模型理解到了：

它们表达的是相似语义

本质上 BERT 学到的是：

语义空间（Semantic Space）

而不是关键词匹配。

什么是 Topic Modeling？

Topic Modeling（主题建模）的目标：

自动发现大量文本中“大家在聊什么”

例如财经新闻：

英伟达 AI 芯片
美联储降息
苹果 Vision Pro
原油价格上涨

传统方法：

  • LDA
  • pLSA

本质依赖：

词频统计

问题也很明显：

  • 不懂语义
  • 同义词效果差
  • 短文本效果很烂

BERTopic 的核心思想

BERTopic 本质是：

BERTopic = embedding + clustering + keyword extraction

它是一个 NLP Pipeline。 而不是一个 Foundation Model。

为什么名字里有BERT? 因为BERTopic 最早提出时：bert embedding非常火，当时作者核心思想是：“用 BERT embedding 做 Topic Modeling”，但是现代BERTopic 已经不依赖原始 BERT，通常更像如下：

现代BERTopic ≈ Embedding-Based Topic Modeling

BERTopic 做的事情其实非常直接：

用 embedding 替代传统词频

整体流程：

文本
 ↓
embedding
 ↓
UMAP降维
 ↓
HDBSCAN聚类
 ↓
c-TF-IDF提取关键词

BERTopic 的工作流程

1. embedding

首先：Sentence-Transformers 或者 BERT 会把文本变成向量：

"CUDA ecosystem keeps growing"
↓
768维向量

语义相近的句子：会在向量空间中靠近。

2. UMAP 降维

高维空间聚类困难。

于是 UMAP 负责：

768维 -> 5维

保留主要语义结构。

3. HDBSCAN 聚类

接着 HDBSCAN 会把语义接近的文本自动聚在一起。例如：

GPU
CUDA
AI芯片
NVIDIA

可能被聚成同一个 Topic。

4. c-TF-IDF 提取主题词

最后 BERTopic 会从聚类结果中提取最能代表该主题的关键词。

例如：

TopicKeywords
AI芯片GPU, CUDA, NVIDIA
宏观经济FED, Inflation, Rate

BERTopic 最厉害的地方：语义联想

例如：

“Huang Renxun talked about AI computing”

里面没有：

  • NVIDIA
  • GPU
  • CUDA

但 embedding 仍然可能认为：

它属于 AI芯片 Topic

因为模型在预训练时已经学到：

黄仁勋 ≈ NVIDIA ≈ GPU ≈ AI芯片

这就是：

语义泛化（Semantic Generalization）

但 BERTopic 不是“真正理解世界”

它本质仍然是：

统计意义上的语义接近

它会：

  • 聚类
  • 联想
  • 建立语义邻近

但不会：

  • 真正推理
  • 创造知识
  • 凭空生成不存在的关键词

例如：

BERTopic 不会凭空生成 “NVIDIA ecosystem”

除非文本里真的出现过。

一个最小实验

下面这个例子非常适合直观理解 BERTopic：

from bertopic import BERTopic
from sklearn.cluster import KMeans
from sklearn.decomposition import PCA

# ===== 示例文本 =====
docs = [
    # AI/GPU
    "NVIDIA released a new GPU",
    "CUDA ecosystem keeps growing",
    "AI chips demand is exploding",
    "Huang Renxun talked about AI computing",

    # 宏观经济
    "The Federal Reserve may cut interest rates",
    "Inflation pressure is decreasing",
    "US bond yields are falling",
    "The economy may enter a soft landing",

    # 苹果产品
    "Apple released Vision Pro",
    "The new iPhone sales are strong",
    "Tim Cook introduced new devices",
    "Apple ecosystem continues expanding",

    # Other
    "Deep learning accelerators are becoming popular"
]

# ===== 创建 BERTopic =====
# macOS/conda/arm64 环境下，BERTopic 默认的 UMAP 会报错，所以我使用的是PCA代替UMAP
topic_model = BERTopic(
    language="english",
    umap_model=PCA(n_components=5),
    hdbscan_model=KMeans(n_clusters=3, random_state=42),
    calculate_probabilities=False,
    verbose=True
)

# ===== 训练 =====
topics, probs = topic_model.fit_transform(docs)

# ===== 输出 Topic =====
print("\n===== 每条文本对应的 Topic =====")
for doc, topic in zip(docs, topics):
    print(f"[Topic {topic}] {doc}")

# ===== 输出 Topic 关键词 =====
print("\n===== Topic Keywords =====")
topic_info = topic_model.get_topic_info()
# 对 Representative_Docs 做截断，避免输出过长
topic_info["Representative_Docs"] = topic_info["Representative_Docs"].apply(
    lambda docs: [
        doc[:20] + "..." if len(doc) > 20 else doc
        for doc in docs
    ]
)

print(topic_info.to_string())

# ===== 查看某个 Topic 的关键词 =====
for topic_id in set(topics):
    print(f"\n===== Topic {topic_id} =====")
    print(topic_model.get_topic(topic_id))

BERTopic 的真正核心

很多人以为 BERTopic 的核心是 BERT。其实真正核心是：

embedding + clustering

现代 BERTopic 往往使用：

  • sentence-transformers
  • bge
  • OpenAI embedding

而不是原始 BERT。因为真正决定 语义能力 的是 embedding 模型。

SentenceTransformer

SentenceTransformer 为什么比原始 BERT 更适合做 embedding？

很多人第一次接触 Sentence-Transformers 时都会疑惑：

“BERT 不已经能输出向量了吗？”

问题在于：

原始 BERT 并不是为“句子相似度”设计的

它的训练目标是：

Masked Language Modeling

例如：

"The cat [MASK] on the table"

预测：

sat

所以它更擅长：

词级别上下文理解

而不是：

句子之间的语义距离

SentenceTransformer 的核心改进

SentenceTransformer 最核心的思想：

专门优化“句子 embedding”

它通常采用：

Siamese Network（孪生网络）

结构。

即：

句子A -> BERT
句子B -> BERT
↓
计算 cosine similarity
↓
反向传播优化

训练目标变成：

相似句子距离更近
不同句子距离更远

例如：

句子A句子B目标
我喜欢篮球我热爱打球接近
我喜欢篮球今天股市暴跌远离

这其实就是：

Contrastive Learning（对比学习）

为什么它效果更强？

因为它真正优化了：

embedding space

于是：

“NVIDIA released a GPU”

和：

“CUDA ecosystem keeps growing”

即使没有重复词：

也可能拥有非常接近的 cosine similarity。

Pooling：另一个容易被忽视的关键

很多人直接使用：

BERT 的 [CLS]

作为句向量。

但 SentenceTransformer 通常会做：

Mean Pooling

即：

对所有 token embedding 求平均

因为：

整个句子的语义
不应该只依赖单个 CLS token

这其实是一个非常重要的工程优化。

为什么现代 NLP 几乎都在使用 SentenceTransformer

因为很多系统本质都依赖：

高质量 embedding

例如：

  • RAG
  • 向量数据库
  • BERTopic
  • 语义搜索
  • 推荐系统
  • 多模态检索

本质上都在做：

embedding space engineering

而 SentenceTransformer：正是现代 embedding 体系中最经典的一类方案。
