MCP 深度解析：让 AI 真正连接世界的协议

系列文章：本文是「MCP 与 Skill 技术体系」三篇系列的第一篇，聚焦于 MCP（Model Context Protocol）的原理、架构与实战。

一、为什么需要 MCP？

大语言模型（LLM）本质上是一个无状态的文本预测系统。它知道如何”思考”，但天生不知道”现在几点了”、“数据库里有什么”、“用户的文件长什么样”。

早期的解法是 Function Calling（函数调用）。以 OpenAI 为代表的厂商在 API 层面允许开发者定义一组工具，让模型在需要时”喊出”工具名称和参数，再由应用程序执行并返回结果。这套方案简单有效，但存在明显局限：

  • 协议碎片化：OpenAI、Anthropic、Google 各自定义了不兼容的 tool schema；
  • 集成成本高：每个 AI 应用都要重新实现工具管理、上下文注入、权限控制；
  • 可复用性差：一个”查询 GitHub Issues”的工具在不同应用里要写三遍。

2024 年 11 月，Anthropic 发布了 Model Context Protocol（MCP），试图用一套统一、开放的标准来解决上述问题。MCP 的野心是成为 AI 领域的 LSP（Language Server Protocol）——就像 LSP 让编辑器与语言服务解耦一样，MCP 让 AI Host 与外部世界解耦。

二、MCP 的核心架构

2.1 三角关系：Host、Client、Server

flowchart TB subgraph Host["Host（宿主）"] Runtime["LLM Runtime / Agent"] <--> Client["MCP Client<br/>(协议实现层)"] end Runtime <--> Provider["模型服务<br/>(Anthropic/OpenAI/本地模型)"] Client -- "MCP Protocol (JSON-RPC 2.0)" --> Server["MCP Server<br/>Tools | Resources | Prompts"] Server --> FS["文件系统 / DB"] Server --> API["外部 API / 服务"]
角色职责实例
Host运行 LLM，管理 MCP Client 生命周期Claude Desktop、VS Code Copilot
MCP Client维护与 Server 的长连接，转发协议消息内嵌在 Host 中的协议适配层
MCP Server暴露工具/资源/提示词，执行具体逻辑mcp-server-github、mcp-server-filesystem

关键设计决策：Client 与 Server 是 1:1 的关系，但一个 Host 可以同时连接多个 Server。这使得工具可以在不同 Server 间隔离，互不影响。

2.2 传输层：两种通信方式

MCP 支持两种传输机制，均基于 JSON-RPC 2.0：

Stdio 传输（本地进程）

Host Process
    │
    ├── stdin  ──────────► MCP Server Process
    └── stdout ◄────────── MCP Server Process

适用于本地工具（文件系统、代码执行器）。Server 以子进程形式运行，通过标准输入输出交换 JSON-RPC 消息。

HTTP + SSE 传输（远程服务）

MCP Client  ──POST /messages──►  MCP Server (HTTP)
MCP Client  ◄──SSE events──────  MCP Server (HTTP)

适用于远程 API 封装。Client 通过 HTTP POST 发送请求，Server 通过 Server-Sent Events 推送响应，天然支持流式输出。

三、MCP 的三大原语

3.1 Tools（工具）——让 AI 能”做事”

Tools 是 MCP 最核心的能力，对应 Function Calling 的概念，但有更完善的描述规范。

JSON Schema 定义示例：

{
	"name": "query_database",
	"description": "执行 SQL 查询并返回结果。仅支持 SELECT 语句，禁止修改操作。",
	"inputSchema": {
		"type": "object",
		"properties": {
			"sql": {
				"type": "string",
				"description": "要执行的 SELECT SQL 语句"
			},
			"database": {
				"type": "string",
				"enum": ["production", "analytics"],
				"description": "目标数据库"
			}
		},
		"required": ["sql", "database"]
	}
}

Python Server 实现（使用 MCP 官方 Python SDK）：

这里的“官方”指 MCP 开源项目组织 modelcontextprotocol 维护的 SDK：

  • MCP快速入门: https://mcp.meetcoding.cn/
  • Python SDK（GitHub）：https://github.com/modelcontextprotocol/python-sdk
  • MCP 官方文档：https://modelcontextprotocol.io/
  • MCP 规范（Specification）：https://modelcontextprotocol.io/specification/latest

原生 MCP Server API（基础）

from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextContent
import sqlite3

app = Server("database-server")

@app.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(
            name="query_database",
            description="执行 SQL 查询并返回结果",
            inputSchema={
                "type": "object",
                "properties": {
                    "sql": {"type": "string"},
                    "database": {"type": "string", "enum": ["production", "analytics"]}
                },
                "required": ["sql", "database"]
            }
        )
    ]

@app.call_tool()
async def call_tool(name: str, arguments: dict) -> list[TextContent]:
    if name == "query_database":
        sql = arguments["sql"]
        # 安全校验：只允许 SELECT
        if not sql.strip().upper().startswith("SELECT"):
            return [TextContent(type="text", text="错误：只允许执行 SELECT 语句")]

        db_path = f"/data/{arguments['database']}.db"
        conn = sqlite3.connect(db_path)
        cursor = conn.execute(sql)
        rows = cursor.fetchall()
        columns = [desc[0] for desc in cursor.description]

        result = {"columns": columns, "rows": rows}
        return [TextContent(type="text", text=str(result))]

async def main():
    async with stdio_server() as (read_stream, write_stream):
        await app.run(read_stream, write_stream, app.create_initialization_options())

if __name__ == "__main__":
    import asyncio
    asyncio.run(main())

# tools.py
import platform
import psutil
import subprocess
import json

def get_host_info() -> str:
    """get host information
    Returns:
        str: the host information in JSON string
    """
    info: dict[str, str] = {
        "system": platform.system(),
        "release": platform.release(),
        "machine": platform.machine(),
        "processor": platform.processor(),
        "memory_gb": str(round(psutil.virtual_memory().total / (1024**3), 2)),
    }

    cpu_count = psutil.cpu_count(logical=True)
    if cpu_count is None:
        info["cpu_count"] = "-1"
    else:
        info["cpu_count"] = str(cpu_count)

    try:
        cpu_model = subprocess.check_output(
            ["sysctl", "-n", "machdep.cpu.brand_string"]
        ).decode().strip()
        info["cpu_model"] = cpu_model
    except Exception:
        info["cpu_model"] = "Unknown"

    return json.dumps(info, indent=4)

if __name__ == '__main__':
    print(get_host_info())

3.2 Resources（资源）——让 AI 能”看到”

Resources 允许 Server 向 Host 暴露结构化的上下文数据，类似于文件或数据库记录。与 Tools 的主动调用不同，Resources 是被动读取的。

from mcp.types import Resource, TextResourceContents

@app.list_resources()
async def list_resources() -> list[Resource]:
    return [
        Resource(
            uri="file:///project/README.md",
            name="项目 README",
            description="项目的主要说明文档",
            mimeType="text/markdown"
        ),
        Resource(
            uri="db://users/schema",
            name="用户表结构",
            description="数据库用户表的 Schema 定义",
            mimeType="application/json"
        )
    ]

@app.read_resource()
async def read_resource(uri: str) -> TextResourceContents:
    if uri == "file:///project/README.md":
        with open("/project/README.md") as f:
            return TextResourceContents(uri=uri, mimeType="text/markdown", text=f.read())
    elif uri == "db://users/schema":
        schema = {"table": "users", "columns": ["id", "name", "email", "created_at"]}
        return TextResourceContents(uri=uri, mimeType="application/json", text=str(schema))

3.3 Prompts（提示词模板）——让 AI 能”复用经验”

Prompts 是预定义的、可参数化的提示词片段，帮助 Host 在正确时机注入标准化的工作指令。

from mcp.types import Prompt, PromptArgument, PromptMessage, TextContent

@app.list_prompts()
async def list_prompts() -> list[Prompt]:
    return [
        Prompt(
            name="code_review",
            description="对代码进行专业的安全性和可读性审查",
            arguments=[
                PromptArgument(name="language", description="编程语言", required=True),
                PromptArgument(name="focus", description="审查重点：security/performance/style", required=False)
            ]
        )
    ]

@app.get_prompt()
async def get_prompt(name: str, arguments: dict) -> list[PromptMessage]:
    if name == "code_review":
        language = arguments.get("language", "unknown")
        focus = arguments.get("focus", "overall")
        return [
            PromptMessage(
                role="user",
                content=TextContent(
                    type="text",
                    text=f"请对以下 {language} 代码进行 {focus} 维度的专业审查，"
                         f"指出潜在问题并给出改进建议："
                )
            )
        ]

四、完整协议流程：一次工具调用的生命周期

1. 初始化阶段
   Client → Server: initialize { protocolVersion, capabilities }
   Server → Client: initialize result { serverInfo, capabilities }
   Client → Server: initialized (通知)

2. 工具发现阶段
   Client → Server: tools/list {}
   Server → Client: { tools: [...] }
   ↓
   Host 将 tools schema 注入 LLM system prompt

3. 推理阶段
   User → LLM: "查询 production 数据库中最近 10 个用户"
   LLM → Host: tool_call { name: "query_database", arguments: { sql: "SELECT ...", database: "production" } }

4. 执行阶段
   Host → Client → Server: tools/call { name, arguments }
   Server 执行 SQL → 返回结果
   Server → Client → Host: { content: [{ type: "text", text: "..." }] }

5. 最终回复
   Host 将工具结果注入对话上下文
   LLM 生成最终自然语言回复 → User

五、MCP 的安全设计

MCP 协议在设计时将安全作为一等公民考虑：

5.1 能力声明与最小权限

Server 在初始化时声明自己的能力集，Client 只暴露必要的工具给 LLM。这实现了最小权限原则——LLM 不会看到它不需要的工具。

5.2 用户授权模型

LLM 请求调用工具
      ↓
Host 拦截 → 展示给用户："AI 想要执行 DELETE FROM users WHERE id=123，是否允许？"
      ↓ (用户确认)
Client 转发给 Server 执行

规范明确要求：对于具有副作用的操作，Host 应在执行前征得用户同意。

5.3 输入验证

# Server 端必须严格验证输入

七、自定义 MCP Server 完整示例

下面是一个完整的本地文件检索 MCP Server，可供 Copilot 调用：

# file: mcp_server_search.py
import asyncio
import os
import re
from pathlib import Path
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextContent

app = Server("code-search-server")

@app.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(
            name="grep_code",
            description="在指定目录中搜索包含特定模式的代码文件",
            inputSchema={
                "type": "object",
                "properties": {
                    "pattern": {
                        "type": "string",
                        "description": "搜索的正则表达式模式"
                    },
                    "directory": {
                        "type": "string",
                        "description": "搜索根目录路径"
                    },
                    "extensions": {
                        "type": "array",
                        "items": {"type": "string"},
                        "description": "限制搜索的文件扩展名，如 ['.py', '.js']"
                    }
                },
                "required": ["pattern", "directory"]
            }
        )
    ]

@app.call_tool()
async def call_tool(name: str, arguments: dict) -> list[TextContent]:
    if name == "grep_code":
        pattern = arguments["pattern"]
        directory = arguments["directory"]
        extensions = arguments.get("extensions", [])

        # 安全检查：防止路径遍历
        root = Path(directory).resolve()
        results = []

        try:
            regex = re.compile(pattern)
        except re.error as e:
            return [TextContent(type="text", text=f"正则表达式错误: {e}")]

        for filepath in root.rglob("*"):
            if not filepath.is_file():
                continue
            if extensions and filepath.suffix not in extensions:
                continue

            try:
                content = filepath.read_text(encoding="utf-8", errors="ignore")
                for i, line in enumerate(content.splitlines(), 1):
                    if regex.search(line):
                        results.append(f"{filepath}:{i}: {line.strip()}")
            except (OSError, PermissionError):
                continue

        if not results:
            return [TextContent(type="text", text="未找到匹配结果")]

        output = "\n".join(results[:100])  # 限制返回行数
        if len(results) > 100:
            output += f"\n... 共 {len(results)} 条结果，仅显示前 100 条"

        return [TextContent(type="text", text=output)]

async def main():
    async with stdio_server() as (read_stream, write_stream):
        await app.run(read_stream, write_stream, app.create_initialization_options())

if __name__ == "__main__":
    asyncio.run(main())

九、FastMCP 快速开发指南

FastMCP 是 MCP Python SDK 中的高级 API，大幅简化了 MCP Server 的开发。相比低级 API，FastMCP 提供了装饰器式的开发体验，让你用最少的代码快速搭建生产级的 MCP 服务。

9.1 FastMCP 的基本用法

最小示例：

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("MyServer")

@mcp.tool()
def add(a: int, b: int) -> int:
    """Add two numbers."""
    return a + b

@mcp.resource("greeting://{name}")
def get_greeting(name: str) -> str:
    """Greet someone."""
    return f"Hello, {name}!"

@mcp.prompt()
def review_code(code: str, language: str = "python") -> str:
    """Code review prompt."""
    return f"Review this {language} code:\n\n{code}"

if __name__ == "__main__":
    mcp.run()

运行方式：

# 直接运行
python my_server.py

# 使用 uv 运行
uv run my_server.py

# 通过 MCP Inspector 测试（包含 UI）
uv run mcp dev my_server.py

# 安装到 Claude Desktop
uv run mcp install my_server.py

9.2 Tool Schema 的查看与调试

FastMCP 会根据函数签名自动生成 JSON Schema，你可以通过几种方式查看：

方式 1：MCP Inspector（推荐）

# 终端 1：启动服务器
uv run mcp dev your_server.py

# 浏览器自动打开 Inspector，可视化查看所有 Tool、Resource、Prompt 及其 Schema

Inspector 会显示：

  • ✅ Tool 的输入 Schema、描述、返回类型
  • ✅ Resource 的 URI 模板、Schema
  • ✅ Prompt 的参数及验证规则
  • ✅ 你可以直接在 UI 中调用 Tool 进行测试

方式 2：通过代码打印

from mcp.server.fastmcp import FastMCP

mcp = FastMCP("Debug")

@mcp.tool()
def query(sql: str, database: str = "default") -> dict[str, str]:
    """Execute a SQL query.

    This tool only supports SELECT statements for safety.
    """
    return {"result": "data"}

# 列出所有 Tool 并查看 Schema
if __name__ == "__main__":
    import asyncio

    async def inspect():

9.3 FastMCP 的常见坑与注意事项

坑 1：Tool 返回类型必须 JSON 序列化

❌ 错误：

@mcp.tool()
def bad_tool() -> set:
    return {1, 2, 3}  # set cannot be JSON serialized

✅ 正确：

@mcp.tool()
def good_tool() -> list:
    return [1, 2, 3]

@mcp.tool()
def good_tool_dict() -> dict[str, int]:
    return {"count": 3}

支持的返回类型：

  • 基础类型：str, int, float, bool, None
  • 容器：list, dict, tuple
  • Pydantic 模型：BaseModel 子类
  • TypedDict
  • 自定义类（需要 __init__ 和类型注解）

坑 2：Async 函数与 Context 的配合

@mcp.tool()
async def async_tool(name: str) -> str:
    """Async tools work fine."""
    return f"Hello, {name}!"

# 如果需要 Context（获取日志、请求元数据等）
from mcp.server.fastmcp import Context
from mcp.server.session import ServerSession

@mcp.tool()
async def tool_with_context(query: str, ctx: Context[ServerSession, None]) -> str:
    """Tool that uses context."""
    await ctx.info(f"User query: {query}")  # Log to client
    return f"Answer to: {query}"

注意：Context 参数必须标注类型，名称可以任意，FastMCP 会自动注入。

坑 3：Tool 的 inputSchema 嵌套对象

当参数过多时，直接用很多参数不如用 Pydantic 模型：

# ❌ 不推荐：函数参数爆炸
@mcp.tool()
def create_user(name: str, email: str, age: int, phone: str,
                address: str, city: str, country: str) -> dict:
    pass

# ✅ 推荐：用 Pydantic 组织参数
from pydantic import BaseModel, Field

class UserInfo(BaseModel):
    name: str = Field(description="User name")
    email: str = Field(description="Email address")
    age: int = Field(ge=0, le=150)
    phone: str | None = None
    address: str | None = None
    city: str = "Unknown"
    country: str = "Unknown"

@mcp.tool()
def create_user(user: UserInfo) -> dict:
    """Create a new user."""
    return {"id": 123, "name": user.name}

坑 4：Resource URI 的模板语法

@mcp.resource() 的 URI 遵循 RFC 6570 URI Template 标准：

# ✅ 正确：使用 {param} 定义变量
@mcp.resource("file://{path}")
def read_file(path: str) -> str:
    return open(path).read()

@mcp.resource("db://table/{table_name}/row/{row_id}")
def get_row(table_name: str, row_id: int) -> dict:
    return {"data": []}

# ❌ 错误：不要用其他格式
# @mcp.resource("file://:[path]")  # 错误

坑 5：Prompt 的参数验证

Prompt 的参数必须有默认值或标记为 required=True：

@mcp.prompt()
def code_review(
    code: str,                          # 使用默认空字符串
    language: str = "python",           # 有默认值
    focus: str = "security"             # 有默认值
) -> str:
    """Code review prompt."""
    return f"Language: {language}, Focus: {focus}\n\n{code}"

坑 6：Lifespan 与资源初始化

如果需要在启动时初始化数据库、连接池等，使用 lifespan：

from contextlib import asynccontextmanager
from typing import AsyncIterator
from dataclasses import dataclass

@dataclass
class AppContext:
    db_connection: str
    shared_cache: dict

@asynccontextmanager
async def app_lifespan(server: FastMCP) -> AsyncIterator[AppContext]:
    # Startup
    print("Initializing resources...")
    db = "connected_db"
    cache = {}
    ctx = AppContext(db, cache)

    try:
        yield ctx
    finally:
        # Cleanup
        print("Closing resources...")

mcp = FastMCP("MyApp", lifespan=app_lifespan)

# 在 Tool 中访问初始化的资源
@mcp.tool()
def query_db(sql: str, ctx: Context) -> dict:
    """Query database using initialized connection."""
    app_context = ctx.request_context.lifespan_context
    db = app_context.db_connection
    cache = app_context.shared_cache
    return {"result": "data"}

坑 7：Tool 的输出 Schema 验证

Tool 可以声明 outputSchema 来约束返回值（需要低级 API）。FastMCP 中如果返回类型是 Pydantic 模型，会自动生成 schema 并验证：

from pydantic import BaseModel, Field

class QueryResult(BaseModel):
    total: int = Field(ge=0, description="Total count")
    rows: list[dict] = Field(default_factory=list, description="Result rows")
    duration_ms: float = Field(ge=0, description="Query duration in ms")

@mcp.tool()
def query(sql: str) -> QueryResult:
    """Execute query and return structured result."""
    # FastMCP 会验证返回值是否符合 QueryResult 的 schema
    return QueryResult(total=100, rows=[{"id": 1}], duration_ms=12.5)

坑 8：错误处理与用户提示

Tool 抛出异常会返回给 AI，建议返回有意义的错误信息：

@mcp.tool()
def safe_query(database: str, table: str) -> str:
    """Query a table safely."""
    ALLOWED_DBS = {"production", "staging"}
    ALLOWED_TABLES = {"users", "products"}

    # Input validation
    if database not in ALLOWED_DBS:
        raise ValueError(f"Database must be one of {ALLOWED_DBS}")

    if table not in ALLOWED_TABLES:
        raise ValueError(f"Table must be one of {ALLOWED_TABLES}")

    # Execute query
    return f"SELECT * FROM {database}.{table}"

9.4 FastMCP 与低级 API 的对比

特性FastMCP低级 API
开发速度⚡ 很快相对较慢
装饰器支持✅ 是❌ 否
自动 Schema 生成✅ 是手动定义
类型检查✅ 完整需要手动处理
灵活性中等⭐⭐⭐ 很高
学习曲线平缓陡峭

选择指南：

  • 快速原型或简单 Server → FastMCP
  • 需要完全自定义或高性能 → 低级 API

十、总结

MCP 的本质是一套标准化的 AI-工具接口协议，它解决的核心问题是：

  1. 互操作性：一个 MCP Server 能被任意支持 MCP 的 Host 使用；
  2. 关注点分离：工具的实现与 AI 的对话逻辑完全解耦；
  3. 安全可控：协议层面内置了权限声明和用户授权机制；
  4. 可组合性：多个 Server 可以同时运行，能力可以叠加。

如果说 Function Calling 是”AI 会打电话”，那 MCP 就是”AI 接入了标准电话网络”——任何电话机都能用，任何号码都能拨。
