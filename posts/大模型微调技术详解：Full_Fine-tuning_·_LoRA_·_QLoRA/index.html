大模型微调技术详解：Full Fine-tuning · LoRA · QLoRA

从原理到代码，彻底搞懂三种主流大模型微调方案的区别与使用方法

背景：为什么需要微调？

当我们使用 LLaMA、Qwen、ChatGLM 这类预训练大模型时，它们已经具备强大的通用能力，但在特定领域任务（比如医疗问答、法律文书、客服对话）上效果往往不够理想。

这时候就需要微调（Fine-tuning）——在特定数据集上继续训练，让模型适应特定任务。

graph LR A["海量通用文本<br/>网页/书籍/代码<br/>1T+ tokens"] -->|预训练| B["基础大模型<br/>通用理解能力强<br/>但领域任务弱"] B -->|微调| C["专属领域模型<br/>医疗/法律/客服<br/>针对任务效果大幅提升"] style A fill:#161b22,stroke:#30363d,color:#58a6ff style B fill:#161b22,stroke:#d2a8ff,color:#d2a8ff style C fill:#161b22,stroke:#3fb950,color:#3fb950

图 1：预训练 → 微调 → 专属模型的流程

微调的核心问题是：大模型参数量巨大（7B ~ 70B 甚至更多），如何在有限资源下完成训练？由此产生了以下三种主流方案。

全量微调（Full Fine-tuning）

全量微调就是最朴素的思路：把模型的所有参数都更新一遍。就像重新教一个人从头到脚改变习惯。

原理：在特定数据集上，通过反向传播（backprop）更新模型的全部参数 W，使 Loss 最小化。

graph TB subgraph Full_Finetuning["全量微调 - 每一层权重 W 全部参与更新"] L1["Layer 1 W1 4096x4096<br/>梯度更新 16,777,216 个参数"] L2["Layer 2 W2<br/>梯度更新"] LN["Layer N Wn<br/>梯度更新"] end Note["模型所有参数均参与反向传播<br/>显存峰值 = 参数量 x 参数+梯度+优化器状态 x 字节数"] L1 --> L2 --> LN style L1 fill:#1c2128,stroke:#ff7b72,color:#ff7b72 style L2 fill:#1c2128,stroke:#ff7b72,color:#ff7b72 style LN fill:#1c2128,stroke:#ff7b72,color:#ff7b72 style Note fill:none,stroke:none,color:#8b949e

图 2：全量微调中每一层的权重矩阵都被更新（红色虚线框表示”参与训练”）

代码示例

# 全量微调示例 (以 LLaMA-3-8B 为例)
from transformers import AutoTokenizer, AutoModelForCausalLM, TrainingArguments, Trainer
from datasets import load_dataset
import torch

# 1. 加载模型（全部参数，fp16）
model_id = "meta-llama/Meta-Llama-3-8B"
tokenizer = AutoTokenizer.from_pretrained(model_id)
model     = AutoModelForCausalLM.from_pretrained(
    model_id,
    torch_dtype=torch.float16,
    device_map="auto",
)

# 2. 查看参数量
total = sum(p.numel() for p in model.parameters())
trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
print(f"总参数量：{total/1e9:.2f}B")
print(f"可训练参数：{trainable/1e9:.2f}B")
print(f"可训练比例：{trainable/total*100:.1f}%")

# 3. 训练配置（全量微调需要极大显存）
args = TrainingArguments(
    output_dir="./full_finetuned",
    num_train_epochs=3,
    per_device_train_batch_size=1,     # 受限于显存
    gradient_accumulation_steps=16,
    learning_rate=2e-5,
    fp16=True,
    save_strategy="epoch",
    logging_steps=10,
)
trainer = Trainer(model=model, args=args, train_dataset=dataset)
trainer.train()

问题所在：全量微调 8B 模型需要约 160GB 显存（参数 + 梯度 + Adam 优化器状态），普通玩家根本跑不起来。这正是 LoRA 和 QLoRA 诞生的动机。

LoRA — Low-Rank Adaptation

LoRA（2021，微软）的核心洞察是：微调时权重矩阵的变化量 ΔW 是低秩的。换句话说，不需要更新整个庞大的权重矩阵，只需要两个小矩阵的乘积来近似这个变化。

核心原理

graph LR X["x 输入"] --> S1["分叉"] X --> S2["分叉"] S1 -->|上路| W["W 冻结不训练<br/>4096x4096"] S2 -->|下路| A["A 训练<br/>4096x8"] A --> B["B 训练<br/>8x4096"] W --> ADD["+"] B --> ADD ADD --> H["h = Wx + BAx"] style X fill:#161b22,stroke:#30363d,color:#8b949e style W fill:#1c2128,stroke:#484f58,color:#484f58 style A fill:#1c2128,stroke:#58a6ff,color:#58a6ff style B fill:#1c2128,stroke:#58a6ff,color:#58a6ff style ADD fill:none,stroke:none,color:#e3b341 style H fill:#1c2128,stroke:#3fb950,color:#3fb950

LoRA 数学公式：

  • h = W₀x + ΔWx
  • ΔW = B · A
  • A ∈ ℝ^(d×r), B ∈ ℝ^(r×d)
  • r << d，r 即 rank（秩）
  • 参数节省比 = 2rd / d² = 2r/d

图 3：LoRA 结构图，W 冻结不动，只训练旁路的 A、B 两个小矩阵

原矩阵参数：4096×4096 = 16.7M → LoRA 参数：2×4096×8 = 65K（减少 99.6%）

关键参数 rank（r）：rank 越大，表达能力越强，但参数量也越多。通常取 4、8、16、64。大多数任务 r=8 或 r=16 已经足够。

代码示例

# 需要安装：pip install transformers peft datasets accelerate
from transformers import AutoTokenizer, AutoModelForCausalLM
from peft import LoraConfig, get_peft_model, TaskType
import torch

model_id = "meta-llama/Meta-Llama-3-8B"
tokenizer = AutoTokenizer.from_pretrained(model_id)
model     = AutoModelForCausalLM.from_pretrained(
    model_id,
    torch_dtype=torch.float16,
    device_map="auto",
)

# ─────────────────────────────────────────────
# 配置 LoRA 参数
# ─────────────────────────────────────────────
lora_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,   # 因果语言模型
    r=8,                            # rank，低秩矩阵的秩
    lora_alpha=32,                 # 缩放系数 α，通常设为 2r
    lora_dropout=0.1,              # dropout 防过拟合
    target_modules=[               # 在哪些层加 LoRA
        "q_proj", "v_proj",        # 注意力的 Q、V 投影
        "k_proj", "o_proj",
    ],
)

# 把 LoRA 插入模型
model = get_peft_model(model, lora_config)

# 查看参数量变化
model.print_trainable_parameters()

from transformers import TrainingArguments, Trainer, DataCollatorForSeq2Seq
from datasets import load_dataset

# 准备数据集（以 alpaca 格式为例）
dataset = load_dataset("tatsu-lab/alpaca", split="train[:1000]")

def preprocess(example):
    prompt = f"### Instruction:\n{example['instruction']}\n\n### Response:\n{example['output']}"
    return tokenizer(prompt, truncation=True, max_length=512)

dataset = dataset.map(preprocess)

# 训练参数（相比全量微调更温和）
args = TrainingArguments(
    output_dir="./lora_output",
    num_train_epochs=3,
    per_device_train_batch_size=4,      # batch size 更大了
    gradient_accumulation_steps=4,
    learning_rate=3e-4,                  # LoRA 通常用更大的 lr
    fp16=True,
    logging_steps=10,
    save_strategy="epoch",
    report_to="none",
)

trainer = Trainer(
    model=model,
    args=args,
    train_dataset=dataset,
    data_collator=DataCollatorForSeq2Seq(tokenizer, model=model, padding=True),
)
trainer.train()

# 保存 LoRA adapter（只需几十 MB！）
model.save_pretrained("./lora_adapter")

QLoRA — Quantized LoRA

QLoRA（2023，华盛顿大学）在 LoRA 基础上更进一步，解决了”即便 LoRA 也还是需要在 fp16 下加载模型”的问题。

量化原理

量化（Quantization） 是指把权重从高精度（fp16/fp32）压缩到低精度（int8/int4），以减少显存占用。

graph LR A["FP32 全精度<br/>32 bit<br/>精度极高"] -->|减半| B["FP16 半精度<br/>16 bit<br/>显存减半 常用于训练"] B -->|量化| C["INT8 8位整型<br/>8 bit<br/>精度略降 推理常用"] C -->|极致压缩| D["NF4 4位 QLoRA用<br/>4 bit<br/>显存减少75%!"] style A fill:#1c2128,stroke:#ff7b72,color:#ff7b72 style B fill:#1c2128,stroke:#ffa657,color:#ffa657 style C fill:#1c2128,stroke:#e3b341,color:#e3b341 style D fill:#1c2128,stroke:#3fb950,color:#3fb950

图 4：不同量化精度对显存的影响（以 8B 参数模型为例）

精度显存 (8B模型)说明
FP32~32 GB全精度
FP16~16 GB半精度，训练常用
INT8~8 GB8位量化
NF4~4 GBQLoRA 核心创新

NF4 (NormalFloat4) 是 QLoRA 提出的专门适合正态分布权重的量化格式

QLoRA 的完整流程 = NF4 量化加载基础模型 + BF16 精度训练 LoRA adapter，两件事合起来做：

graph LR S1["Step 1<br/>NF4 量化<br/>基础模型 4bit 压缩<br/>16GB 约 4GB"] --> S2["Step 2<br/>插入 LoRA<br/>在量化模型上<br/>加 A B 小矩阵"] S2 --> S3["Step 3<br/>BF16 训练<br/>只更新 LoRA 参数<br/>W 保持 4bit 冻结"] S3 --> R["微调完成<br/>约 6 GB<br/>单卡可跑!"] style S1 fill:#161b22,stroke:#e3b341,color:#e3b341 style S2 fill:#161b22,stroke:#58a6ff,color:#58a6ff style S3 fill:#161b22,stroke:#d2a8ff,color:#d2a8ff style R fill:#161b22,stroke:#3fb950,color:#3fb950

图 5：QLoRA 完整训练流程

QLoRA 让在单张 RTX 4090 (24GB) 上微调 70B 模型成为可能

代码示例

# 需要安装：pip install transformers peft bitsandbytes accelerate trl
from transformers import (
    AutoTokenizer, AutoModelForCausalLM,
    BitsAndBytesConfig,              # ← 量化配置
    TrainingArguments,
)
from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training
from trl import SFTTrainer
import torch

model_id = "meta-llama/Meta-Llama-3-8B"

# ─────────────────────────────────────────────
# Step 1：配置 4bit 量化（NF4）
# ─────────────────────────────────────────────
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,              # 开启 4bit 量化
    bnb_4bit_quant_type="nf4",       # 使用 NF4 格式
    bnb_4bit_use_double_quant=True,  # 嵌套量化，再省一点显存
    bnb_4bit_compute_dtype=torch.bfloat16,  # 计算时用 bf16
)

# Step 2：加载量化后的模型
model = AutoModelForCausalLM.from_pretrained(
    model_id,
    quantization_config=bnb_config,  # ← 传入量化配置
    device_map="auto",
)
tokenizer = AutoTokenizer.from_pretrained(model_id)
tokenizer.pad_token = tokenizer.eos_token

# Step 3：为 kbit 训练做准备（处理梯度检查点等）
model = prepare_model_for_kbit_training(model)

# Step 4：插入 LoRA
lora_config = LoraConfig(
    r=16,
    lora_alpha=64,
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj",
                    "gate_proj", "up_proj", "down_proj"],  # MLP 层也加
    lora_dropout=0.05,
    bias="none",
)
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()

# Step 5：训练
args = TrainingArguments(
    output_dir="./qlora_output",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=2,
    gradient_checkpointing=True,   # 省显存
    learning_rate=2e-4,
    bf16=True,                       # bf16 更稳定
    warmup_ratio=0.03,
    logging_steps=10,
    save_strategy="epoch",
)

trainer = SFTTrainer(
    model=model,
    args=args,
    train_dataset=dataset,
    dataset_text_field="text",
    tokenizer=tokenizer,
    max_seq_length=512,
)
trainer.train()
