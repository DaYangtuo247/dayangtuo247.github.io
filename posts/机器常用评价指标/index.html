机器学习常用评价指标

**混淆矩阵（Confusion Matrix）**以及常用评价指标，按照你要求的“横轴是真实，纵轴是预测”来说明。为了直观理解，我会从二分类问题开始，再扩展到多分类情况。

1. 混淆矩阵定义

混淆矩阵是用于评价分类模型性能的一种工具。它用一个矩阵来表示模型预测结果与真实标签的对应关系，能直观看出模型在哪些类别上预测正确，在哪些类别上犯了错误。

1.1 二分类情况

假设有两个类别：正类 (Positive, P) 和 负类 (Negative, N)，并且按照你的要求：

  • 横轴（列）是真实类别
  • 纵轴（行）是预测类别

矩阵可以表示为：

真实正类 (P)真实负类 (N)
预测正类 (P’)TPFP
预测负类 (N’)FNTN

解释：

  • TP（True Positive）：真实为正类，被模型预测为正类 → 正确预测正类
  • TN（True Negative）：真实为负类，被模型预测为负类 → 正确预测负类
  • FP（False Positive）：真实为负类，但被预测为正类 → 假阳性（Type I Error）
  • FN（False Negative）：真实为正类，但被预测为负类 → 假阴性（Type II Error）

2. 常用评价指标

基于混淆矩阵，我们可以定义一些常用指标：

2.1 准确率（Accuracy）

表示预测正确的比例，公式：

Accuracy=TP+TNTP+TN+FP+FN\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}Accuracy=TP+TN+FP+FNTP+TN​
  • 含义：模型整体上预测对的比例。
  • 优缺点：在类别不平衡时，准确率可能会有误导性（比如负类占大多数，即使模型总是预测负类，准确率也高）。

2.2 精确率（Precision，也叫查准率）

表示预测为正类中，真实为正类的比例，公式：

Precision=TPTP+FP\text{Precision} = \frac{TP}{TP + FP}Precision=TP+FPTP​
  • 含义：模型预测为正时，有多大概率是真的正类。
  • 适用场景：当误报（FP）代价高时（比如垃圾邮件过滤，把正常邮件误判为垃圾邮件）。

2.3 召回率（Recall，也叫查全率、True Positive Rate）

表示真实为正类中，被预测为正类的比例，公式：

Recall=TPTP+FN\text{Recall} = \frac{TP}{TP + FN}Recall=TP+FNTP​
  • 含义：模型能找到多少真实正类。
  • 适用场景：当漏报（FN）代价高时（比如疾病检测，漏掉患者是很严重的）。

2.4 F1-score

精确率和召回率的调和平均，公式：

F1=2⋅Precision⋅RecallPrecision+RecallF1 = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}F1=2⋅Precision+RecallPrecision⋅Recall​
  • 含义：兼顾精确率和召回率的综合指标。
  • 适用场景：类别不平衡或需要权衡 FP 和 FN 时。

2.5 特异性（Specificity）

表示真实为负类中，被预测为负类的比例，公式：

Specificity=TNTN+FP\text{Specificity} = \frac{TN}{TN + FP}Specificity=TN+FPTN​
  • 含义：负类预测正确的能力。
  • 与召回率互补：Recall 是对正类的敏感度，Specificity 是对负类的敏感度。

2.6 假阳性率（False Positive Rate, FPR）

表示实际为负类，被误预测为正类的比例，公式：

FPR=FPFP+TN=1−SpecificityFPR = \frac{FP}{FP + TN} = 1 - \text{Specificity}FPR=FP+TNFP​=1−Specificity
  • 含义：负类被误报为正类的概率。

3. 多分类情况

如果类别有 (C) 类，可以扩展混淆矩阵为 (C \times C) 的矩阵：

  • 行：预测类别
  • 列：真实类别
  • 每个单元格 (i,j)：表示预测为类别 (i)，真实为类别 (j) 的样本数。

然后可以对每个类别计算：

  • 单类别的 TP、FP、FN、TN
  • 计算多分类的 宏平均（Macro Average） 或 微平均（Micro Average） 精度、召回率、F1-score。

4. 可视化示意

比如二分类，矩阵表示：

           真实正 (P)   真实负 (N)
预测正(P')     50         10
预测负(N')     5          35
  • TP = 50
  • FP = 10
  • FN = 5
  • TN = 35

指标：

  • Accuracy = (50+35)/100 = 0.85
  • Precision = 50/(50+10) ≈ 0.833
  • Recall = 50/(50+5) ≈ 0.909
  • F1 ≈ 0.87

5. PR 曲线（Precision-Recall Curve）

定义：

  • PR 曲线展示的是 精确率（Precision） 与 召回率（Recall） 在不同阈值下的关系。
  • 横轴：召回率（Recall）
  • 纵轴：精确率（Precision）

注意：这是与混淆矩阵相关的指标衍生出来的曲线，主要用于正类关注度较高的任务（不平衡数据集）。

5.1 计算方式

分类模型（如逻辑回归、神经网络）通常输出一个概率分数 (p)：

  1. 设定阈值 (t)，大于 (t) 判定为正类，否则判定为负类。
  2. 根据混淆矩阵计算 Precision 和 Recall。
  3. 改变阈值 (t)（从 0 到 1），得到一系列 (Recall, Precision) 点。
  4. 绘制这些点，连接起来就是 PR 曲线。

5.2 PR 曲线特点

  • 当数据类别不平衡时，PR 曲线比 ROC 曲线更有参考价值，因为它重点关注正类的预测性能。
  • 理想情况：曲线靠近 右上角（Recall ≈ 1，Precision ≈ 1）。

6. PR 曲线下的指标

6.1 平均精度（Average Precision, AP）

  • 对 PR 曲线下的 精确率取积分，衡量曲线下面积。
  • AP 越高，说明模型在不同阈值下，正类预测效果越好。

公式（离散化形式）：

AP=∑n(Rn−Rn−1)PnAP = \sum_n (R_n - R_{n-1}) P_nAP=n∑​(Rn​−Rn−1​)Pn​

其中 (R_n) 和 (P_n) 是第 n 个阈值下的 Recall 和 Precision。

6.2 F1-score 与 PR 曲线

  • 在 PR 曲线上，每个点都对应一个阈值下的 F1-score： F1=2⋅P⋅RP+RF1 = 2 \cdot \frac{P \cdot R}{P + R}F1=2⋅P+RP⋅R​
  • 可以通过曲线上找到 F1-score 最大值的点，作为最佳阈值。

7. 与 ROC 曲线对比

指标PR 曲线ROC 曲线
横轴Recall（召回率）FPR（False Positive Rate）
纵轴Precision（精确率）TPR（True Positive Rate / Recall）
关注点正类预测质量（不平衡数据好用）分类器整体区分能力
曲线高分区右上角左上角

直观理解：PR 曲线更“挑剔”，强调模型预测正类的可靠性；ROC 曲线更“宽松”，强调模型区分正负类的能力。

8. 使用场景

  • PR 曲线：异常检测、疾病预测、金融欺诈检测（正类样本少，但重要性高）。
  • ROC 曲线：类别比较均衡的分类问题（如猫狗识别）。

9. 可视化示意

  1. X 轴：Recall（召回率）0 → 1
  2. Y 轴：Precision（精确率）0 → 1
  3. 曲线从右上角开始（高 Precision，低 Recall），随着阈值降低，Recall 增加，Precision 通常下降。
  • 最高点对应 低阈值 → 模型预测尽量多的正类（Recall 高，但可能 Precision 低）
  • 最低点对应 高阈值 → 模型只预测最有信心的正类（Precision 高，但 Recall 低）
