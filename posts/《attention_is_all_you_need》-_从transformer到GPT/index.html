《attention is all you need》阅读笔记

从 Transformer 到 GPT：Encoder、Decoder、Self-Attention、Masked Self-Attention 全面理解

这是一篇面向初学者的笔记，目标不是堆公式，而是用尽量直观的方式，把 Transformer、Encoder、Decoder、Cross-Attention、Masked Self-Attention 以及 GPT-2 的结构关系讲清楚。 如果你刚开始学习《Attention Is All You Need》，看这篇会更容易建立整体框架。

本文章参考多处来源（包括但不限于），如有侵权，请邮件/评论联系作者，列举如下：

  • Transformer Explainer CN
  • 菜鸟教程 Transformer 结构
  • Hand-Drawn Transformer

1. 为什么要写这篇笔记

我在学习 Transformer 的时候，最开始其实有很多困惑：

  • Encoder 和 Decoder 到底分别在干什么？
  • 为什么翻译任务要用完整 Transformer？
  • 为什么 GPT-2 只需要 Decoder？
  • Masked Self-Attention 为什么是上三角 mask？
  • 训练阶段到底是怎么训练的？是不是一步一步预测？
  • 推理阶段是不是就不需要 Mask 了？
  • Cross-Attention 中的 Q、K、V 到底分别来自哪里？

这些问题如果只看论文结构图，其实很容易“似懂非懂”。 所以这篇文章的目标，是把这些问题串起来，形成一套完整、统一的理解。

2. Transformer 整体结构：先看大图景

原始 Transformer（《Attention Is All You Need》）是一个 Encoder-Decoder 架构。左边是 Encoder，右边是 Decoder。

它的经典应用场景是：

  • 机器翻译
  • 摘要生成
  • 问答中的 Seq2Seq 任务
  • 语音转文本等“输入序列 → 输出序列”的任务

可以先粗略理解为：

  • Encoder：负责“理解输入”
  • Decoder：负责“生成输出”

例如翻译任务：

  • 输入：英文句子
  • 输出：法文句子

这时：

  • Encoder 读英文
  • Decoder 写法文

所以 Transformer 不是单纯“接收一句话然后继续往下写”，而是更像：

先把输入这段话理解成一份内部表示，再根据这份表示去生成另一段输出。 而现代的 GPT-2 则是“只保留了 Decoder 部分”，它的任务是： 给定前文，预测后文。所以它不需要单独的 Encoder 来理解输入，而是直接在同一个序列里做自回归预测。

3. Encoder 和 Decoder 分别在干什么

3.1 Encoder 的作用：编码输入

假设输入句子是：

I love you

Encoder 会把这三个 token 编码成对应的向量表示。 这些表示不是简单的词向量，而是 融合了上下文信息后的表示。

也就是说，Encoder 输出的不是孤立的 I、love、you，而是：

  • “这个 I 出现在当前句子里的语义”
  • “这个 love 在当前上下文下的语义”
  • “这个 you 在当前上下文下的语义”

所以 Encoder 的输出，本质上可以理解为：

对输入句子的语义压缩和表示

这些输出随后会提供给 Decoder 使用。

3.2 Decoder 的作用：生成输出

Decoder 的任务不是“直接输出答案”，而是：

根据已经生成的内容 + Encoder 提供的输入信息，预测下一个 token 的概率分布。

这点很重要。

例如翻译时目标句子是：

Je t’aime

Decoder 不会一下子整句吐出来，而是这样：

  1. 先输入 <BOS>（开始符）
  2. 预测第一个 token：Je
  3. 再输入 <BOS> Je
  4. 预测第二个 token：t’
  5. 再输入 <BOS> Je t’
  6. 预测第三个 token：aime

所以 Decoder 是 自回归生成 的。

4. Self-Attention 到底是什么

Transformer 的核心机制是 Attention。

最经典的公式是：

Attention(Q,K,V)=softmax(QKTdk)V\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)VAttention(Q,K,V)=softmax(dk​​QKT​)V

初看这个公式会很抽象，但可以先这样理解：

  • Q（Query）：当前想关注什么
  • K（Key）：每个位置提供“索引”
  • V（Value）：每个位置真正携带的信息

Attention 做的事情就是：

用当前的 Query 去和所有 Key 做匹配，得到权重，再对所有 Value 做加权求和。

通俗一点说：

当前 token 会问：“我应该重点看序列中的哪些位置？”

5. Encoder 里的 Self-Attention

Encoder 中使用的是普通的 Self-Attention。

所谓 Self-Attention，就是：

  • Q 来自当前序列
  • K 来自当前序列
  • V 来自当前序列

也就是：

Q = 输入序列本身
K = 输入序列本身
V = 输入序列本身

例如句子：

I love you

在 Encoder 里，每个 token 都可以看见整个句子：

  • I 可以看 love
  • love 可以看 you
  • you 也可以看前面的内容

所以 Encoder 的 Self-Attention 是 双向的、全局的。

这也是为什么 Encoder 非常适合理解输入。

6. Decoder 里有两种 Attention，不是一种

这是学习 Transformer 时最容易混淆的点之一。

Decoder 里其实有 两层注意力机制：

  1. Masked Self-Attention
  2. Cross-Attention

6.1 第一层：Masked Self-Attention

这一层的 Q、K、V 都来自 Decoder 当前序列本身：

Q = decoder 当前序列
K = decoder 当前序列
V = decoder 当前序列

但是和 Encoder 不同的是，它带有 Mask。

这个 Mask 的作用是：

禁止当前位置看见未来位置。

也就是说，在预测下一个词时，模型不能偷看后面的标准答案。

6.2 第二层：Cross-Attention

这一层才是 Decoder “参考输入”的关键所在。

它的来源关系是：

Q = 来自 decoder
K = 来自 encoder
V = 来自 encoder

这意味着：

  • Decoder 先根据自己当前生成状态形成 Query，其实就是将经过masked self-attention + add & Norm之后的输出（恰好是768维，参考gpt2），直接投影成一个新的 Query，这个 Query 代表了当前生成状态的语义信息。
  • 然后拿这个 Query 去查询 Encoder 输出的 Key/Value
  • 从而决定当前生成时应该关注原句的哪一部分

例如翻译：

I love you
→ Je t’aime

当 Decoder 在生成 aime 时，需要知道它对应原句里的 love。 Cross-Attention 就是在做这种“对齐”和“参考”。

7. 为什么翻译任务要用完整 Transformer

这个问题很关键。

7.1 因为翻译是 Seq2Seq 问题

翻译不是简单的“给定前文预测后文”，而是：

英文序列 → 中文/法文序列

输入和输出是 两个不同序列，甚至还是两种不同语言。

所以需要明确地区分：

  • 输入是什么
  • 输出是什么

这就是 Encoder-Decoder 架构存在的意义：

  • Encoder 专门理解原文
  • Decoder 专门生成译文
  • Cross-Attention 负责让生成过程能够参考原文

7.2 如果没有 Encoder 会怎样

如果把输入和输出全揉在一起，模型当然也不是完全不能做翻译，但会有一些问题：

  • 输入和输出边界不清晰
  • 参考原文的机制不够明确
  • 结构上不够适合“输入输出分离”的任务

所以原始 Transformer 用 Encoder + Decoder，是一种很自然、很适合翻译的设计。

8. 为什么 GPT-2 只需要 Decoder

GPT-2结构可视化 transformer explainer cn

这就要看 GPT-2 做的任务是什么。

GPT-2 的核心任务不是翻译，而是 语言建模（Language Modeling）：

给定前面的 token，预测下一个 token。

也就是说，它做的是：

x_1, x_2, ..., x_t → 预测 x_{t+1}

这个任务里，输入和输出其实属于同一个序列。 模型不需要“读一段源序列，再生成另一段目标序列”，而是只需要：

不断根据已有上下文继续往下写。

所以 GPT-2 不需要：

  • Encoder
  • Cross-Attention

只需要 Decoder 中的：

  • Masked Self-Attention
  • FFN
  • 残差连接
  • LayerNorm

即可。

所以 GPT-2 可以理解为：

只保留了 Transformer 的 Decoder 部分，并去掉了 Cross-Attention。

9. GPT-2 为什么不需要 Cross-Attention

因为 GPT 的所有上下文都已经放在同一个序列里了。

比如对话场景中，模型看到的输入可能是：

系统提示 + 历史对话 + 当前问题

这些内容全部拼接成一个长序列，然后模型继续预测下一个 token。

此时模型只需要：

  • 用当前 token 去关注前面所有 token
  • 决定下一个最可能输出什么

也就是说，GPT 并不需要一套单独的 Encoder 来提供 K/V。 它直接在同一个序列内部做 Masked Self-Attention 就够了。

10. Masked Self-Attention 为什么能防止“看见未来”

这是很多人刚学 Transformer 时最不直观的点。

10.1 先看普通 Attention 矩阵

假设有 4 个 token：

[A, B, C, D]

计算 QK^T 后，会得到一个分数矩阵，大致可以想象成：

      A   B   C   D
A     •   •   •   •
B     •   •   •   •
C     •   •   •   •
D     •   •   •   •

这里：

  • 行表示当前 Query 所在位置
  • 列表示可以关注的 Key 所在位置

如果不加限制，那么 A 可以看到 B、C、D， B 可以看到 C、D， 这就会导致“看未来”。

10.2 加上 Mask 后会怎样

我们人为构造一个上三角遮挡矩阵：

      A    B    C    D
A     0   -∞   -∞   -∞
B     0    0   -∞   -∞
C     0    0    0   -∞
D     0    0    0    0

这表示：

  • 第 1 个位置不能看后面 3 个位置
  • 第 2 个位置不能看后面 2 个位置
  • 第 3 个位置不能看后面 1 个位置
  • 最后一个位置没有未来可看

然后把这个 Mask 加到原本的注意力分数上：

softmax(QKT+mask)\text{softmax}(QK^T + \text{mask})softmax(QKT+mask)

由于 -∞ 进入 softmax 后，概率会变成 0， 所以这些“未来位置”的注意力权重会被强制清零。

例如：softmax([1, 2, -∞]) → [0.27, 0.73, 0]

这就是为什么说：

Masked Self-Attention 能阻止模型看到未来。

10.3 为什么是上三角

因为序列是从左到右排列的：

  • 左边是过去
  • 右边是未来

所以在注意力矩阵里，“未来”刚好对应右上区域。 把右上区域设为 -∞，就等于屏蔽未来。

所以常见说法“使用上三角 mask”，本质上是：

把未来位置的注意力权重强制置零。

11. Masked Self-Attention 在训练时是怎么工作的

很多初学者会以为训练时是“一步一步预测、一步一步更新参数”。 其实不是。

11.1 训练时是整段文本一次性输入

例如一句话：

I love you

训练时通常做成：

输入：I love you
目标：love you <EOS>

也就是 Shift Right 后的目标形式。

模型会一次性对所有位置并行计算：

  • P(love | I)
  • P(you | I, love)
  • P(EOS | I, love, you)

注意，虽然是一次性并行算的，但由于有 Mask，每个位置只能看到它前面的内容。

所以它在逻辑上等价于：

每个位置都在做“预测下一个词”

11.2 不是逐行跑，而是并行跑

这一点很重要。

RNN 是真正一步一步串行地算。 而 Transformer 训练时是：

整个序列一次性输入，然后所有位置并行计算。

Mask 保证了这种并行计算不会作弊。

所以可以说：

Mask 让“并行训练”模拟出了“逐步预测”的效果。

12. 训练阶段是不是需要 temperature=0

不需要。

这是训练和推理最容易混淆的地方之一。

12.1 训练时不采样

训练时模型输出的是每个位置的 概率分布，例如：

[0.1, 0.7, 0.2]

真实标签可能是第 2 个词。 这时不会去“采样一个 token 看看对不对”，而是直接计算损失：

loss=−log⁡P(正确词)\text{loss} = -\log P(\text{正确词})loss=−logP(正确词)

也就是交叉熵损失。

所以训练时关心的是：

正确 token 的概率有多大

而不是：

模型随机抽出来的 token 是不是正好等于正确答案

12.2 temperature 只在推理时用

Temperature 是生成阶段的采样控制参数，用来调节输出的随机性：

  • temperature 小：更确定，更接近 argmax
  • temperature 大：更随机，更有创造性

训练阶段一般不使用 temperature，也不做采样。

13. 训练阶段完整流程

以 Decoder-only 模型为例，训练流程可以概括成：

13.1 输入序列

x = [x_1, x_2, x_3, ..., x_n]

13.2 构造目标

输入：x_1, x_2, ..., x_{n-1}
目标：x_2, x_3, ..., x_n

13.3 前向传播

模型一次性输出每个位置对下一个 token 的预测分布。

13.4 计算损失

把每个位置预测出的分布与对应真实 token 计算交叉熵损失，然后求和或求平均。

13.5 反向传播

根据总损失更新参数。

这些参数不仅包括：

  • W_q
  • W_k
  • W_v

还包括：

  • 输出投影矩阵
  • FFN 层参数
  • Embedding
  • LayerNorm 参数等

所以不能简单理解为“只训练 QKV”。

14. 推理阶段是不是还要经过完整网络

是的，逻辑上必须经过完整网络。

14.1 对于 Encoder-Decoder 模型

以翻译为例：

训练阶段

  • 输入原文进 Encoder
  • 正确译文（Shift Right）进 Decoder
  • Decoder 通过 Masked Self-Attention + Cross-Attention 生成预测
  • 计算 loss，反向传播

推理阶段

  • 原文仍然要先进 Encoder
  • Decoder 从 <BOS> 开始
  • 每次生成一个 token
  • 再把已生成 token 接回去继续生成下一个

也就是说，推理时并不是“Decoder 输入为空”，而是：

Decoder 从 <BOS> 开始，逐步构建输出序列。

14.2 对于 GPT 这类 Decoder-only 模型

训练阶段：

  • 整段文本一次性输入
  • 用 Masked Self-Attention 并行预测所有位置的下一个 token

推理阶段：

  • 输入上下文
  • 每次生成一个 token
  • 再把新 token 拼回上下文
  • 继续生成

14.3 为什么推理通常更慢

因为训练时可以并行算所有位置， 而推理时第 t+1 个 token 必须等第 t 个 token 先生成出来。

所以推理天然是串行的。

当然，工程上会使用 KV Cache 优化，但那是“复用历史计算”，不是改变结构本身。

15. Cross-Attention 中的长度问题：Q 变长了，K/V 不变怎么办

这是一个很经典的问题。

Cross-Attention 中：

  • Q 来自 Decoder
  • K/V 来自 Encoder

假设：

  • Encoder 输入长度为 n
  • Decoder 当前已经生成长度为 m

那么张量形状可以写成：

  • (Q \in \mathbb{R}^{m \times d})
  • (K \in \mathbb{R}^{n \times d})
  • (V \in \mathbb{R}^{n \times d})

注意力分数矩阵就是：

QKT∈Rm×nQK^T \in \mathbb{R}^{m \times n}QKT∈Rm×n

这意味着：

Decoder 中的每一个位置，都去关注 Encoder 中的所有位置。

所以不要求 m=n。 Q 和 K 的序列长度不同是完全正常的。

这也解释了为什么 Decoder 已经生成很长一句话时，仍然可以去参考固定长度的原文表示。

16. Decoder 中到底有没有“自己的 KV”

有，而且有两套逻辑要区分清楚。

16.1 Masked Self-Attention 中有 Decoder 自己的 KV

这里：

Q = decoder
K = decoder
V = decoder

这是 Decoder “理解自己已经生成了什么”的部分。

16.2 Cross-Attention 中没有 Decoder 的 KV

这里：

Q = decoder
K = encoder
V = encoder

这是 Decoder “参考输入”的部分。

所以不要把整个 Decoder 里所有 Attention 都看成一套 KV。 实际上：

  • 第一层 Attention 用 Decoder 自己的 KV
  • 第二层 Attention 用 Encoder 提供的 KV

17. 一个很容易犯的错误：层之间传递的不是 Q/K/V，而是隐藏表示

很多初学者会误以为：

Masked Self-Attention 先算出一个 Q，然后把这个 Q 交给 Cross-Attention。

这不准确。

更准确的说法是：

每一层 Attention 都会根据当前输入重新投影出一套新的 Q/K/V。

也就是说，层之间传递的是隐藏状态 h，而不是原封不动的 Q/K/V。

流程更像是：

  1. 输入隐藏状态 h
  2. 线性变换得到 Q/K/V
  3. 做 Attention
  4. 得到新的隐藏状态 h'
  5. 再输入到下一层

所以：

Q/K/V 是每层内部临时构造出来的视角，而不是跨层传递的数据实体。

18. 如何统一理解 Transformer 和 GPT

到这里，其实可以把两者统一起来看。

18.1 原始 Transformer

适合：

  • 翻译
  • 摘要
  • 语音识别
  • 多模态输入输出任务

特点：

  • 输入和输出是两个不同序列
  • 使用 Encoder 处理输入
  • 使用 Decoder 生成输出
  • 使用 Cross-Attention 连接两者

18.2 GPT / GPT-2

适合：

  • 文本续写
  • 对话
  • 代码生成
  • 通用语言建模

特点：

  • 输入和输出属于同一条序列
  • 只有 Decoder
  • 没有 Encoder
  • 没有 Cross-Attention
  • 依靠 Masked Self-Attention 完成自回归预测

19. 一套最终的心智模型

如果只记一套最重要的理解，我会建议记下面这个版本。

19.1 Encoder 在做什么

把输入句子编码成一组上下文相关的语义表示。

19.2 Decoder 在做什么

根据已经生成的内容，结合输入信息，预测下一个 token。

19.3 Masked Self-Attention 在做什么

保证当前位置只能看到过去，不能看到未来，从而学会真正的自回归预测。

19.4 Cross-Attention 在做什么

让 Decoder 在生成输出时，能够有选择地参考 Encoder 对输入的编码结果。

19.5 GPT 为什么只需要 Decoder

因为 GPT 的任务是“给定前文，预测后文”，输入和输出本来就在同一个序列里，不需要再单独建一个 Encoder。

20. 结语

Transformer 看起来复杂，其实拆开后就几个核心点：

  1. Attention：决定该关注哪里
  2. Encoder：理解输入
  3. Decoder：生成输出
  4. Masked Self-Attention：防止看未来
  5. Cross-Attention：生成时参考输入
  6. GPT：只保留 Decoder，用于自回归语言建模

很多困惑并不是因为概念太难，而是因为：

  • 训练和推理容易混淆
  • Self-Attention 和 Cross-Attention 容易混淆
  • 原始 Transformer 和 GPT 容易混淆

只要把这三组区别理顺，整个结构就会清晰很多。
