Cloudflare 加速 + GitHub Pages + 阿里云域名

参考文章：

  1. GitHub pages+自定义域名(腾讯云域名)+cloudflare加速
  2. Github Pages 域名更换 + 基于 Cloudflare 加速国内访问

一、架构与域名规划

  • www.wqlblog.cn：博客主站（走 Cloudflare 代理）
  • wqlblog.cn：可选裸域（301 到 www）
  • api.wqlblog.cn：评论/统计 API（指向 Cloudflare Worker）
  • dayangtuo247.github.io：GitHub Pages 默认域名（保留）

api.wqlblog.cn 主要用于评论系统（如 waline）和统计服务（访问量、评论数、点赞数），通过 Cloudflare Worker + D1 database 实现，但是国内无法访问worker.dev域名，所以需要通过 自定义域名来实现这个功能，并且这个域名也需要走 Cloudflare 代理，否则国内访问会非常慢。 参考文章：Cloudflare Workers + 国内优化

二、GitHub Pages 基础配置

这里需要注意，由于域名是阿里云买的，所以DNS解析得在阿里云做。

有两个很坑的点：

  1. cloudflare中先不要配置DNS加速
  2. 不能在cloudflare中做CNAME记录，必须在阿里云，在cloudflare中配置，GitHub page始终无法通过DNS Check

做好之后，回到github page，等待DNS检查通过，之后在去cloudflare中开启DNS加速

cn域名必须要使用国内注册的域名管理（如阿里云）提供的DNS服务，否则无法解析

证据如下，顶级域名的DNS仍然是阿里云的DNS服务器，二级域名的DNS才是Cloudflare的DNS服务器；

所以CNAME记录必须添加在阿里云的DNS服务器上，而不是Cloudflare的DNS服务器上。
