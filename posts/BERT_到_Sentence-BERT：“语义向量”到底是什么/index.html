BERT 到 Sentence-BERT：“语义向量”到底是什么

参考资料：

  • 读懂BERT，看这一篇就够了
  • 词向量 | Transformer & Bert | 原理简介+代码简析

很多人第一次学习 BERT 的时候，都会有一种很强的困惑：

BERT 到底做了什么？ 为什么它改变了 NLP？ 为什么最后输出的是向量？ CLS 为什么能代表整个句子？ Sentence-BERT 又是什么？

我一开始也有类似的问题。后来真正理解之后，我发现：

Transformer/BERT/GPT 的核心，其实是“构建语义空间”。

这篇文章不会只讲论文定义，而是从“输入 → embedding → attention → contextual embedding → sentence embedding”这条主线，彻底理解 BERT 的本质。

transformer、BERT、GPT的关系

一、BERT 到底做了什么？

BERT 的本质是：利用上下文，给每个 token 构建“动态语义向量”。

例如：

苹果很好吃

和：

苹果发布了新Mac

虽然都有：苹果

但：第一句是水果，第二句是公司

传统 embedding 永远不变。如：

embedding("苹果") = [0.2, 0.7, 0.1]

而 BERT：会根据上下文动态变化。于是：

苹果(水果) = [0.9, 0.1, 0.2]
苹果(公司) = [0.1, 0.8, 0.7]

这就是：Contextual Embedding（上下文化向量），也是 BERT 最革命的地方。

二、BERT 的本质结构

BERT 本质上就是 Transformer Encoder，结构非常简单：

输入token
↓
Embedding
↓
Transformer Encoder × N层
↓
输出每个token的语义向量

它只使用了 Transformer 的：Encoder 部分，没有 Decoder。

三、BERT 的输入是什么？

假设输入一句话：

我喜欢AI

Step1：Tokenizer

首先切分 token：

tokens = [
    "[CLS]",
    "我",
    "喜欢",
    "AI",
    "[SEP]"
]

这里：

token含义
[CLS]句子级别 token
[SEP]句子结束

Step2：转换成 token id

token_ids = [
    101,
    2769,
    1599,
    8888,
    102
]

Step3：Embedding

每个 token 查 embedding 表：

embedding("喜欢") = [0.12, -0.33, 0.91, ...]

最终：

embeddings.shape = [seq_len, hidden_size]

例如：

[5, 768]

表示：

  • 5 个 token
  • 每个 token 768维

真实情况

实际上输入还是会添加 position encoding 的，而且更为复杂，还需要添加 sentence encoding 等，这里简化了。

四、Attention 做了什么？

这是 Transformer 最核心的部分。

Attention 本质上：是 token 之间的信息交换

例如：

小明把苹果放桌上，然后他吃了它

这里 它 会重点关注 苹果，因为相关性最高。

Attention 的本质可以理解成：

每个 token 都在问：

“为了理解我自己，我应该关注谁？”

于是：

token embedding
↓
attention 与其他 token 通信
↓
融合上下文
↓
得到 contextual embedding

五、BERT 的输出到底是什么？

很多人以为：BERT 输出分类结果，其实不是。BERT 默认输出：每个 token 的上下文语义向量

例如输入：

[CLS] 我 喜欢 AI [SEP]

输出：

[
  [....],   # CLS
  [....],   # 我
  [....],   # 喜欢
  [....],   # AI
  [....]    # SEP
]

shape：

[seq_len, hidden_size]

例如：

[5, 768]

重点：

  • 输入几个 token

  • 输出几个向量

六、CLS 是什么？

很多教程都会说： CLS 是句子的 embedding

但这里有一个容易误解的地方。

更准确的理解

CLS：是“全局语义聚合点”

例如：

[CLS] 我喜欢AI

Attention 时：

CLS
↕
我
↕
喜欢
↕
AI

所以：CLS 会聚合整句信息。

因此：很多分类任务：

cls_embedding
↓
Linear
↓
分类

就能工作。

七、为什么原始 BERT 的 CLS 不适合句子相似度？

这是很多人第一次接触 embedding 时最困惑的问题。

原始 BERT 的训练目标：

1. MLM（Mask Language Model）

例如：

我今天去了[MASK]

预测：

银行

2. NSP（Next Sentence Prediction）

判断：

句子B 是否是句子A下一句

注意，它从来没训练过：“相似句子向量应该靠近”

所以直接进行cosine(cls1, cls2)，很多时候效果很差。

八、Sentence-BERT 做了什么？

后来：Sentence-BERT 出现了。

它真正解决的问题：如何构建“高质量句向量空间”

Sentence-BERT：不是输出 token embedding。

而是：输出 sentence embedding

例如输入：

我喜欢猫

输出：

sentence_embedding = [0.12, -0.88, 0.45, ...]

重点是这个向量空间有“语义距离意义”

例如：我喜欢猫 和 我爱小猫 会距离很近，而和 今天股票跌停 会 距离很远

九、Mean Pooling 为什么比 CLS 更常见？

工业界做 embedding 很多时候不用 CLS。而是用 Mean Pooling，即：

sentence_embedding = mean(token_embeddings)

原因：每个 token 都已经包含上下文信息。平均后能更稳定表达整句语义

十、BERT 和 Sentence-BERT 的本质区别

模型本质
BERTtoken-level contextual embedding
Sentence-BERTsentence embedding

  • BERT 更像：token 理解器，输出：每个 token 的上下文语义

  • Sentence-BERT 更像：句子编码器，输出：整个句子的语义向量

十一、这和 Speaker Embedding 非常像

这里有一个特别重要的类比。

Speaker Recognition（说话人识别）

输入：

一段声音

模型：

提取说话人特征

输出：

speaker_embedding

于是：

同一个人：

embedding 靠近

不同人：

embedding 远离

Sentence-BERT 本质一样

输入：

一句话

输出：

sentence_embedding

训练目标：

相似句子靠近
不同句子远离

十二、现代 AI 的统一本质

学到这里，会突然发现：很多 AI 模型底层其实是一回事。

输入

复杂数据：

  • 文本
  • 图片
  • 音频
  • 视频

Encoder

Encoder

输出

Embedding Vector

然后：

在向量空间中做：

  • 相似度
  • 检索
  • 聚类
  • rerank
  • 推荐
  • RAG

十三、真正重要的一句话

你现在可以这样理解：

原始 BERT

重点：
理解 token

输出：

token-level contextual embedding

Sentence-BERT

重点：
构建句子语义空间

输出：

sentence embedding

Speaker Encoder

重点：
构建说话人特征空间

输出：

speaker embedding

它们本质上：都是在做“语义编码”。

代码示例

import torch
from transformers import BertTokenizer, BertModel
from sentence_transformers import SentenceTransformer
from sklearn.metrics.pairwise import cosine_similarity

torch.set_printoptions(precision=3, sci_mode=False)

# ======================
# 加载模型
# ======================
tokenizer = BertTokenizer.from_pretrained("bert-base-chinese")

bert = BertModel.from_pretrained(
    "bert-base-chinese",
    output_attentions=True
)

sbert = SentenceTransformer(
    "sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2"
)

bert.eval()

# ======================
# 输入句子
# ======================
sentence = "苹果发布了新的处理器芯片"

print(f"\n原始句子: {sentence}")

# ======================
# tokenizer
# ======================
inputs = tokenizer(
    sentence,
    return_tensors="pt"
)

tokens = tokenizer.convert_ids_to_tokens(
    inputs["input_ids"][0]
)

print("\nTokenizer:")
print(tokens)

# ======================
# 进入 BERT Encoder 前的 embedding
# word embedding + position embedding + token type embedding + LayerNorm
# ======================
with torch.no_grad():
    input_emb = bert.embeddings(
        input_ids=inputs["input_ids"],
        token_type_ids=inputs["token_type_ids"]
    )

print("\n进入 BERT Encoder 前的 Embedding（前4维）:")

for token, emb in zip(tokens, input_emb[0]):
    print(f"{token:>6} -> {emb[:4].tolist()}")

# ======================
# BERT 输出
# ======================
with torch.no_grad():
    outputs = bert(**inputs)

hidden = outputs.last_hidden_state

print("\nBERT 后 Token Contextual Embedding（前4维）:")

for token, emb in zip(tokens, hidden[0]):
    print(f"{token:>6} -> {emb[:4].tolist()}")

# ======================
# CLS
# ======================
cls_emb = hidden[0, 0]

print("\nCLS Embedding（前8维）:")
print(cls_emb[:8])

# ======================
# Mean Pooling
# 注意：这里过滤掉 padding，但当前句子没有 padding
# ======================
attention_mask = inputs["attention_mask"]

mask = attention_mask.unsqueeze(-1).expand(hidden.size()).float()

mean_emb = (hidden * mask).sum(dim=1) / mask.sum(dim=1)

mean_emb = mean_emb[0]

print("\nMean Pooling Sentence Embedding（前8维）:")
print(mean_emb[:8])

# ======================
# Attention
# ======================
print("\nAttention（最后一层，所有 head 平均，过滤特殊 token）:")

# outputs.attentions[-1] shape:
# [batch, num_heads, seq_len, seq_len]
last_layer_attn = outputs.attentions[-1]

# 取第一个样本，然后对所有 head 求平均
# shape: [seq_len, seq_len]
attn = last_layer_attn[0].mean(dim=0)

special_tokens = {"[CLS]", "[SEP]", "[PAD]"}

for i, token in enumerate(tokens):

    if token in special_tokens:
        continue

    scores = attn[i].clone()

    # 过滤掉特殊 token
    for j, t in enumerate(tokens):
        if t in special_tokens:
            scores[j] = -1

    topk = torch.topk(scores, 3)

    print(f"[{token}] 最关注: ", end="")

    for idx, score in zip(topk.indices, topk.values):
        print(f"{tokens[idx]}:{score.item():.3f} ", end="")

    print()

# ======================
# Sentence-BERT
# ======================
sent_emb = sbert.encode(sentence)

print("\nSentence-BERT Embedding（前8维）:")
print(sent_emb[:8])

print("\nSentence-BERT shape:", sent_emb.shape)

# ======================
# 相似度测试
# ======================
s1 = sentence
s2 = "苹果推出人工智能处理器"
s3 = "今天下雨了"

def cls_embedding(text):
    x = tokenizer(
        text,
        return_tensors="pt"
    )

    with torch.no_grad():
        o = bert(**x)

    return o.last_hidden_state[0, 0].numpy()

def mean_pooling_embedding(text):
    x = tokenizer(
        text,
        return_tensors="pt"
    )

    with torch.no_grad():
        o = bert(**x)

    hidden = o.last_hidden_state
    mask = x["attention_mask"].unsqueeze(-1).expand(hidden.size()).float()

    emb = (hidden * mask).sum(dim=1) / mask.sum(dim=1)

    return emb[0].numpy()

def sim(a, b):
    return cosine_similarity([a], [b])[0][0]

# 原始 BERT CLS
b1 = cls_embedding(s1)
b2 = cls_embedding(s2)
b3 = cls_embedding(s3)

print("\n原始 BERT CLS 相似度:")
print(f"相似句({s2}):", sim(b1, b2))
print(f"不相似({s3}):", sim(b1, b3))

# 原始 BERT Mean Pooling
m1 = mean_pooling_embedding(s1)
m2 = mean_pooling_embedding(s2)
m3 = mean_pooling_embedding(s3)

print("\n原始 BERT Mean Pooling 相似度:")
print(f"相似句({s2}):", sim(m1, m2))
print(f"不相似({s3}):", sim(m1, m3))

# Sentence-BERT
e1 = sbert.encode(s1)
e2 = sbert.encode(s2)
e3 = sbert.encode(s3)

print("\nSentence-BERT 相似度:")
print(f"相似句({s2}):", sim(e1, e2))
print(f"不相似({s3}):", sim(e1, e3))
