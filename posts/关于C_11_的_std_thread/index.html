C++11 std::thread

本文详细整理了 C++11 中 std::thread 的核心概念、用法、注意事项以及实践经验，帮助读者深入理解多线程编程。本文在基础内容的基础上，进一步深入探讨线程启动时机、阻塞细节、线程所有权模型、detach 的行为、多线程顺序和调度等高级概念，并配以示例代码和实用建议。

1. 线程基础概念

1.1 线程定义

  • 线程（Thread） 是操作系统能够进行运算调度的最小单位，多个线程共享进程的地址空间和资源。
  • C++11 引入了 std::thread，提供了跨平台的线程支持，封装了底层操作系统线程API。
  • std::thread 对象是线程的“所有权句柄”，负责管理线程的生命周期和状态。

1.2 线程所有权模型

  • std::thread 对象持有线程的“所有权”，即代表对线程的管理权。
  • 线程所有权是独占的，不能被复制，只能移动（通过移动构造或移动赋值）。
  • 当 std::thread 对象销毁时，如果仍持有线程所有权且线程未被 join 或 detach，程序会调用 std::terminate，导致异常终止。
  • 因此，必须在 std::thread 对象析构前调用 join() 或 detach()，确保线程资源得以正确释放。

2. 线程的创建与启动

2.1 线程启动时机

  • 创建 std::thread 对象时，线程函数和参数会立即传入构造函数，线程会立即被操作系统调度启动。
  • 线程启动是异步的，构造函数返回后，线程可能已经执行完毕，也可能尚未开始执行。
  • 线程的调度和执行顺序由操作系统决定，不保证创建顺序即执行顺序。

2.2 示例：创建并启动线程

#include <thread>
#include <iostream>

void worker(int x) {
    std::cout << "Worker thread received: " << x << std::endl;
}

int main() {
    std::thread t(worker, 10); // 线程创建时即启动
    t.join(); // 等待线程执行结束，回收资源
    return 0;
}

2.3 线程对象状态与所有权

  • joinable()：判断当前 std::thread 是否持有一个可连接的线程。
  • 新创建的线程对象默认 joinable() == true。
  • 调用 join() 或 detach() 后，线程对象不再持有线程，joinable() == false。
  • 移动线程对象后，原对象变为不可连接状态。

3. 线程的生命周期管理

3.1 join()

  • 作用：阻塞调用线程，等待目标线程执行完毕，回收线程资源。
  • 调用后，std::thread 对象不再拥有线程（joinable() == false）。
  • 只能调用一次，重复调用会导致程序异常终止（std::terminate）。
  • 阻塞的是调用 join() 的线程，不影响其他线程或整个进程。

3.1.1 阻塞细节

  • join() 会使调用线程进入阻塞状态，等待被 join 的线程执行完毕。
  • 这是同步线程结束的一种方式，保证线程安全地回收资源。
  • 如果目标线程很快结束，join() 调用可能几乎无阻塞。

示例：

std::thread t(worker, 10);
if (t.joinable()) {
    t.join(); // 阻塞等待线程结束
}

3.2 detach()

  • 作用：放弃对线程的管理，让线程独立运行，由操作系统负责线程资源回收。
  • 调用后，std::thread 对象不再关联线程（joinable() == false）。
  • 线程仍属于进程，不会成为孤儿线程，进程退出时所有线程都会被强制终止。
  • 风险：线程访问的外部数据可能已销毁，导致未定义行为。调试困难，建议谨慎使用。

3.2.1 detach 的行为与注意事项

  • detach() 后线程成为后台线程，无法再对其进行同步操作。
  • 线程结束时，系统自动回收资源，无需调用 join()。
  • 由于无法保证线程何时结束，访问共享数据时必须确保数据生命周期安全。
  • 适用于执行不依赖外部状态或不需要通知主线程的后台任务。

示例：

std::thread t(worker, 10);
t.detach(); // 线程独立运行，主线程不等待

3.3 joinable()

  • 判断当前 std::thread 是否持有线程所有权。
  • 返回 true 表示可以调用 join() 或 detach()。
  • 返回 false 表示线程未创建、已 join、已 detach 或被移动。

4. 传递参数给线程函数

4.1 参数传递机制

  • 默认按值传递参数，线程函数接收到的是参数的拷贝。
  • 若传递引用，必须使用 std::ref 或 std::cref 包装，否则会传递引用的拷贝（即指针拷贝），可能导致悬空引用。

4.2 示例：传递引用参数

#include <thread>
#include <iostream>
#include <functional> // std::ref

void increment(int& x) {
    x++;
}

int main() {
    int a = 10;
    std::thread t(increment, std::ref(a)); // 传引用
    t.join();
    std::cout << "a = " << a << std::endl; // 输出 11
    return 0;
}

5. 获取线程执行结果

std::thread 本身不支持直接获取线程函数返回值。常用方法有以下几种：

5.1 使用 std::async（推荐）

  • std::async 创建异步任务，返回 std::future 对象。
  • 通过 future.get() 获取返回值，自动等待任务完成。
  • 适合需要返回结果的异步操作。

示例：

#include <future>
#include <iostream>

int worker(int x) {
    return x * 2;
}

int main() {
    auto fut = std::async(std::launch::async, worker, 10);
    int result = fut.get(); // 阻塞等待结果
    std::cout << "Result: " << result << std::endl; // 输出 20
    return 0;
}

5.2 使用 std::promise + std::future

  • std::promise 用于在线程中设置值。
  • 主线程通过 std::future 获取值。
  • 需要手动管理线程生命周期。

示例：

#include <thread>
#include <future>
#include <iostream>

void worker(std::promise<int> p, int x) {
    int result = x * 3;
    p.set_value(result); // 设置结果
}

int main() {
    std::promise<int> p;
    std::future<int> f = p.get_future();
    std::thread t(worker, std::move(p), 10);
    int result = f.get(); // 阻塞等待结果
    t.join();
    std::cout << "Result: " << result << std::endl; // 输出 30
    return 0;
}

5.3 使用共享变量（不推荐）

  • 线程写全局或共享变量。
  • 需要使用同步机制保证线程安全（如互斥锁）。
  • 容易出错且不优雅。

6. 线程安全与同步

6.1 多线程访问共享数据问题

  • 多线程访问共享数据时，必须保证同步，避免数据竞态和未定义行为。
  • 数据竞态可能导致程序崩溃、数据损坏或不可预测的行为。

6.2 常用同步机制

  • std::mutex 互斥锁：保证同一时刻只有一个线程访问临界区。
  • std::lock_guard：RAII 风格自动加锁解锁，防止死锁。
  • std::condition_variable 条件变量：线程间通知机制。

6.3 示例（互斥锁）

#include <thread>
#include <mutex>
#include <iostream>

int counter = 0;
std::mutex mtx;

void increment() {
    std::lock_guard<std::mutex> lock(mtx);
    ++counter;
}

int main() {
    std::thread t1(increment);
    std::thread t2(increment);
    t1.join();
    t2.join();
    std::cout << "Counter: " << counter << std::endl;
    return 0;
}

7. 线程调度与执行顺序

7.1 线程调度

  • 线程调度由操作系统负责，基于优先级、时间片等策略。
  • C++ 标准不保证线程执行顺序或调度策略。
  • 多线程程序中，线程执行顺序是非确定性的。

7.2 多线程顺序控制

  • 通过同步机制（如互斥锁、条件变量、信号量）控制线程间执行顺序。
  • 例如，使用 std::condition_variable 让线程等待某个条件，保证顺序执行。

7.3 示例：使用条件变量控制顺序

#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>

std::mutex mtx;
std::condition_variable cv;
bool ready = false;

void worker() {
    std::unique_lock<std::mutex> lock(mtx);
    cv.wait(lock, []{ return ready; }); // 等待主线程通知
    std::cout << "Worker thread running after main thread signal." << std::endl;
}

int main() {
    std::thread t(worker);

    {
        std::lock_guard<std::mutex> lock(mtx);
        ready = true;
    }
    cv.notify_one(); // 通知 worker 线程

    t.join();
    return 0;
}

8. 线程状态表

方法作用状态变化注意事项
join()阻塞调用线程，等待线程结束释放所有权，joinable() == false只能调用一次
detach()放弃管理，让线程独立运行释放所有权，joinable() == false线程可能访问已销毁数据，风险大
joinable()判断是否持有线程所有权无返回 true 可调用 join 或 detach

9. 实践建议

  • 优先使用 join() 或 std::async，避免使用 detach()，除非非常确定线程生命周期安全。
  • 使用 joinable() 检查线程状态，避免重复调用 join() 或 detach()。
  • 线程参数尽量传值或用 std::ref 传引用，避免悬空引用。
  • 需要返回值时，优先使用 future 机制获取结果，保证线程安全。
  • 注意线程安全，合理使用同步机制，避免数据竞态。
  • 线程结束后及时回收资源，避免资源泄漏。
  • 避免在线程函数中抛出异常，若必须抛出，应保证异常被捕获，否则会导致程序异常终止。

10. 总结

C++11 的 std::thread 提供了强大的多线程支持，但也带来了复杂的资源管理和同步问题。理解线程的生命周期、参数传递、返回值获取和线程安全是编写正确、高效多线程程序的关键。结合现代 C++ 的异步机制（如 std::async、std::promise 和 std::future），可以更方便地实现并发编程。同时，深入理解线程启动时机、阻塞细节、所有权模型和调度机制，有助于编写更健壮、高效的多线程程序。
