Video-XL-Pro: Reconstructive Token Compression for Extremely Long Video Understanding

May 25, 2026
14 min read
—

introduction

  • 题目：Video-XL-Pro: Reconstructive Token Compression for Extremely Long Video Understanding（面向极长视频理解的重建式Token压缩模型）
  • 会议/期刊：arXiv Preprint（最新预印本，截止2025年）
  • 作者：Xiangrui Liu, Yan Shu, Zheng Liu 等
  • 单位：上海交通大学人工智能学院、北京智源人工智能研究院（BAAI）、特伦托大学

problem defionition

  • 核心问题：现有的多模态大语言模型（MLLMs）在理解**极长视频（超过3分钟，甚至长达数小时）**时面临巨大的性能和算力瓶颈。
  • 问题拆解：
    1. Token爆炸：长视频需要采样更多帧，导致视觉Token数量呈线性增长。
    2. 显存溢出（OOM）：Token数量过多会带来极高的显存开销，超出当前大语言模型的上下文窗口限制。
    3. 信息冗余：视频相邻帧之间存在大量重复的背景和静止画面（类似于量化中高频数据存在大量无意义的微小波动），直接输入给LLM会造成算力浪费并干扰模型注意力。

prior work

针对视频理解，前人的工作主要分为以下三类：

  • 第一类：多帧特征直接拼接法（如早期的Video MLLMs）
    • 思想：用视觉编码器（如CLIP）提取每一帧的特征，然后像串糖葫芦一样拼接起来直接输入LLM。
    • 代表方法：标准LLaVA架构的直接扩展。
    • 缺点：完全无法处理长视频。Token数量极其庞大，模型极易因为超出上下文长度或显存不足而崩溃。
  • 第二类：固定比例的Token压缩法（如基于Q-Former或池化的压缩）
    • 思想：设计专门的模块，强制把每一帧或连续几帧的Token数量压缩到一个固定的比例（例如8倍或16倍下采样）。
    • 代表方法：LLaVA-VID, Chat-UniVi。
    • 缺点：压缩质量不可知且缺乏灵活性。不同视频的“信息密度”不同（静止的监控画面 vs 高频切换的动作电影），固定比例压缩会丢失高信息密度视频的关键细节，且没有客观的标准来评估压缩后的Token质量好坏。
  • 第三类：基于随机掩码（Mask）的自监督学习（视频MAE）
    • 思想：借鉴Masked Autoencoders，随机遮住视频的一部分，让模型去重建，以此学习视频特征。
    • 代表方法：VideoMAE。
    • 缺点：掩码策略盲目。随机遮挡往往会选中毫无信息量的背景，导致模型学不到关键的动态时空特征，而且为了达到好效果需要海量的训练数据，训练成本高昂。

improvement

针对上述痛点，本文提出了一个仅有3B参数的高效模型 Video-XL-Pro。核心创新是引入了**重建式Token压缩（ReCoT）**机制。

针对前人方法的缺点（P2）本文做出的改进与创新点（Video-XL-Pro）
Token爆炸与定长拼接动态Token合成器 (DTS)：不仅做空间压缩，还在时间维度融合，将静态图像Token转化为包含时空动态信息的伪视频Token。
掩码盲目，浪费算力语义引导掩码 (SGM)：不再随机遮挡，而是通过可学习的机制，自适应地识别并掩蔽冗余的视觉Token，让模型专注重建有意义的区域。
长视频训练成本极其高昂视频数据集剪枝策略 (Pruning)：基于帧间相似度剔除冗余视频数据，用更少但更精的高质量数据进行训练。
推理时依然受限于LLM窗口Query感知选择器 (Query-aware Selector)：采用RAG（检索增强）思想，根据用户的提问（Query），动态丢弃无关的视觉Token。

important function

在训练动态Token合成器时，模型需要知道哪些Token重要、哪些不重要（从而去掩蔽它们）。本文提出了一种注意力得分计算公式（对应SGM语义引导掩码）：

  1. 提取当前帧的空间基准特征： Vt←Mean(videofeature,dim=1)V_t \leftarrow Mean(videofeature, dim=1)Vt​←Mean(videofeature,dim=1) (解释：沿着维度1求均值，代表当前帧画面的全局空间语义均值。)
  2. 计算时间维度得分 (Temporal Score)： Stemp←Vt−1⊗TempQueryS_{temp} \leftarrow V_{t-1} \otimes TempQueryStemp​←Vt−1​⊗TempQuery scoretemp←torch.einsum(videofeature,Stemp)score_{temp} \leftarrow torch.einsum(videofeature, S_{temp})scoretemp​←torch.einsum(videofeature,Stemp​) _(解释：利用上一帧的特征 Vt−1V_{t-1}Vt−1​ 和一个可学习的“时间Query矩阵”相乘，再与当前帧特征算内积。这衡量了该Token在时间轴上的动态变化重要性。)_
  3. 计算空间维度得分 (Spatial Score)： Sspa←Vt⊗SpatioQueryS_{spa} \leftarrow V_t \otimes SpatioQuerySspa​←Vt​⊗SpatioQuery scorespa←videofeature⊗Sspascore_{spa} \leftarrow videofeature \otimes S_{spa}scorespa​←videofeature⊗Sspa​ (解释：利用当前帧的全局均值 VtV_tVt​ 和可学习的“空间Query矩阵”计算。这衡量了该Token在当前静态画面中的空间重要性。)
  4. 汇总最终得分： tokenscore=scoretemp+scorespatokenscore = score_{temp} + score_{spa}tokenscore=scoretemp​+scorespa​ (解释：综合时间和空间重要性得出最终得分。训练时，模型会根据这个得分随机掩蔽（Mask）掉那些低分的Token，强迫模型去关注并重建高分（关键）区域。)

framework

  1. 上半部分：重建式Token压缩（ReCoT）的自监督预训练
    • 流程：输入视频 →\rightarrow→ 冻结的SigLIP（提取基础特征） →\rightarrow→ SGM（根据刚才讲的公式进行智能打分和Mask掩蔽） →\rightarrow→ DTS（动态合成压缩） →\rightarrow→ 解码器（试图从被压缩的特征中还原原图像）。
    • 目的：不依赖人工标注，让模型自己学会如何用最少的Token完美代表整个视频的内容。
  2. 下半部分：多模态大模型（MLLM）微调与推理
    • 流程：视频数据集剪枝（去除废数据） →\rightarrow→ SigLIP+DTS提取出高浓度Token →\rightarrow→ 投影层(Projector) →\rightarrow→ Query感知选择器（根据用户提问再次筛选Token） →\rightarrow→ 大语言模型生成答案。

framework detail

结合刚才的架构图，我们将核心步骤拆解为4步：

  1. 特征初始化与动态合成 (DTS模块)： 视频首先经过SigLIP视觉编码器，得到密集的图像Token。随后进入DTS模块，DTS包含时空注意力块和3D卷积，它不仅在空间上将Token压缩了4倍，还将连续4帧的特征在时间维度融合为1帧，使得原本静态的图像Token具备了捕捉动态运动的能力。
  2. 智能掩码与重建 (SGM训练阶段)： 使用前文讲过的评分公式，对Token进行时空重要性打分。对于那些得分低的“背景板”Token，模型将其Mask掉。然后强制要求Decoder用剩下的高分压缩Token还原出完整的原始特征（MSE Loss）。这使得压缩后的Token具备极强的信息代表性。
  3. 视频数据集剪枝 (微调准备阶段)： 长视频训练太耗时。作者对数据按均匀抽样20帧，计算帧内余弦相似度。如果一个视频相似度极高（说明画面毫无变化，高度冗余），则将其从数据集中剔除。这极大提升了后续微调的效率。
  4. Query感知选择器 (推理/问答阶段)： 这是针对极长视频的最后一道防线。当输入Token长度超出大模型（如Qwen2.5）的限制时，模型先将用户的文本提问（Query）通过一个轻量级MLP去跟所有的视觉Token计算注意力得分。直接丢弃掉得分最低的（与问题无关的）Token，保留与问题高度相关的Token输入给LLM进行回答。

experiment

  • 数据集建设：
    • 预训练阶段使用了约200万图文对和经过“视频长度过滤”的视频。
    • 微调阶段（SFT）加入了Bunny、Sharegpt-4o等数据集，并使用提出的数据集剪枝策略进行了清洗。
    • 采用可变帧率采样：对短视频按FPS采样，长视频固定抽样1 FPS（最高支持360帧用于训练）。（详见第5页 Figure 3）。
  • 评测指标：涵盖了主流的长视频理解榜单，如MLVU、Video-MME、LongVideoBench，以及时序定位基准V-STaR和Charades-STA。
  • 实现细节：
    • 基础LLM使用 Qwen-2.5-3B（仅30亿参数）。
    • 视觉编码器结合了冻结的SigLIP和本文训练的DTS。
    • 硬件：32张 A800-80GB GPU，ZeRO-2优化，bf16精度。

result of SOTA

  1. 跨级别碾压：从Table 1可以看出，作为只有3B参数的小个子，Video-XL-Pro在MLVU（得分为70.6%）和Video-MME等核心榜单上，不仅超越了同量级的模型，甚至击败了绝大多数7B级别的开源模型（如Apollo-7B、Video-XL 7B），部分指标比肩GPT-4o。
  2. 效率与精度的完美平衡：如Figure 1所示，在达到SOTA精度的同时，它是GPU显存利用率最优的。
  3. 极长视频“大海捞针”测试：(看Figure 4的绿图)。传统模型（如LLaVA-NeXT）处理几百帧就报OOM内存溢出了。而Video-XL-Pro在单张A100上可以无压力处理超过8192帧的超长视频，且准确率保持在98.85%！

result of ablation

  1. DTS与SGM模块的有效性 (Table 4/5/6)：
    • 相较于原生的SigLIP特征，加入DTS后，即使Token数量被压缩了4倍，模型在视频分类上的精度依然更高。
    • SGM（智能掩码）vs 随机掩码：在同等压缩率下，使用SGM引导掩蔽的模型，在MLVU上提升了约0.7%，证明智能剔除冗余比随机抛弃好得多。
  2. Query感知选择器的超参数动态性能 (Figure 7)： 这个图非常关键！横轴是被过滤掉的Token比例。可以看出，即便是直接丢弃掉50%的Token，模型的性能指标（MLVU等）几乎保持不掉！甚至在丢弃90%的极端情况下，依然保留了相当客观的准确度。这证明基于RAG思想的文本查视觉Token策略极为精准。

conclusion

本论文提出了 Video-XL-Pro，它巧妙地解决了长视频理解中的“显存刺客”问题。

  1. 创新性：通过重建式Token压缩（ReCoT），利用动态合成（DTS）和语义掩码（SGM），实现了视频特征的高效“浓缩”，而不是暴力的下采样。
  2. 工程实用性：配套提出了视频数据集剪枝和Query感知选择器，从训练和推理两端双管齐下降低了算力开销。
  3. 结果极具性价比：仅仅利用 3B的参数量 和少量的训练数据，就做到了单卡跑通8K+帧长视频，打败了7B规模的前沿模型。这种将冗余的时序特征提取出高价值信号的思路，对我们做量化金融中的超长金融时间序列特征压缩与去噪，同样具有很好的启发意义。
