从零实现一个 Transformer（0.023B）：Tokenizer、Mask、训练与可视化

完整代码：https://github.com/DaYangtuo247/transfomer-study

最近在系统地学习 Transformer。以前看论文和结构图时，很多概念都像是“知道有这么回事”，比如 QKV、Masked Self-Attention、KV Cache、Positional Encoding。但真正自己动手写代码之后，才发现很多细节并没有完全理解。

所以这个项目的目标不是训练出一个可以商用的翻译系统，而是从零实现一个简化版 seq2seq Transformer，把下面这条完整链路走通：

文本数据 → Tokenizer → Dataset → Mask → Encoder → Decoder → Loss → 推理 → 可视化

项目使用 AI Challenger 英中翻译语料 的子集，中英文数据各选取前 100w 条（吐槽一下，数据集质量不高，以后训练开始前，一定要先看数据集质量。。。），完成 English -> Chinese 的简单翻译任务。代码保留了较多注释，并加入位置编码、Mask、训练曲线和 Attention 热图可视化，方便理解 Transformer 内部到底在做什么。

训练历史曲线

功能：监控模型训练过程，用于判断模型是否收敛、是否过拟合，以及学习率调度是否正常。

坐标轴含义：

  • X 轴：训练轮次 Epoch。
  • Y 轴：对应指标数值，例如 loss、token accuracy。

如何解读：

  • Loss 是否下降；
  • Token Accuracy 是否上升；
  • 学习率调度是否正常；
  • 是否出现明显过拟合。

Attention 热图

功能：可视化 Transformer 中 Cross-Attention 的权重分布，用于模型可解释性分析。默认图片展示所有 attention heads 的平均值；如果在代码里传入 head=0 等编号，可以查看单个 head。

坐标轴含义：

  • Y 轴 (Predicted Target Token)：当前正在生成的中文 token。
  • X 轴 (Key Position)：输入序列中的词，或者被关注的词。

颜色含义：颜色越深，表示注意力权重越大。也就是说，Y 轴上的词在生成时越依赖 X 轴上的词。

格子数值：每个方格内会直接显示对应的 attention 权重，范围通常在 0 到 1 之间。

如何解读（以机器翻译为例）：

  • 如果输出是 I love you，输入是 Je t'aime。
  • 观察 love 对应的行，应该在 t'aime 对应的列上有高亮，也就是较高注意力权重。
  • 对角线高亮在 Self-Attention 中很常见，表示词更关注自身或邻近词。
  • 在本项目的 English -> Chinese 翻译任务中，横轴是英文源句 token，纵轴是中文目标句 token。它可以帮助你观察模型生成某个中文 token 时，主要关注了哪些英文 token。

作用：调试模型是否学到了正确的对齐关系，或者发现模型是否关注了无关的噪声。

位置编码可视化

功能：展示 Transformer 中正弦 / 余弦位置编码的结构。Transformer 没有 RNN 的时间递推结构，也没有 CNN 的局部窗口结构，所以它本身并不知道 token 的顺序。位置编码就是为了告诉模型：每个 token 在句子中的位置。

生成的图：位置编码矩阵热图和若干维度上的位置编码曲线。

坐标轴含义：

  • 热图 X 轴：embedding 维度。
  • 热图 Y 轴：token 位置。
  • 曲线图 X 轴：token 位置。
  • 曲线图 Y 轴：位置编码数值。

如何解读：不同维度对应不同频率的正弦 / 余弦信号。低频维度变化较慢，高频维度变化较快，使模型可以同时获得局部和全局的位置信息。

Mask 可视化

功能：展示 Transformer 中 padding mask、causal mask 和 decoder mask 的作用，帮助理解模型哪些位置可以看，哪些位置必须屏蔽。

颜色含义：

  • 蓝色区域：可以被 attention 看到的位置。
  • 浅色区域：被 mask 屏蔽的位置。
  • 橙色提示：padding 开始的位置。

三种 mask 的含义：

  • Padding Mask：屏蔽 <pad> 位置，避免模型关注无效 token。
  • Causal Mask：屏蔽未来 token，防止 decoder 在训练时偷看答案。
  • Decoder Mask：padding mask 和 causal mask 的组合，是 decoder self-attention 实际使用的 mask。

Decoder self-attention 中会同时使用二者：

tgt_pad_mask = create_padding_mask(tgt_input, pad_id)
causal_mask = create_causal_mask(tgt_input.size(1))
tgt_mask = tgt_pad_mask & causal_mask

这样可以保证 decoder 既不会关注 padding，也不会偷看未来答案。

train_losses

learning_rates

其中：

  • model_state_dict：模型参数，用于推理和 attention 可视化。
  • optimizer_state_dict / scheduler_state_dict：保存优化器和学习率调度器状态。
  • train_losses / valid_losses / train_accuracies / valid_accuracies / learning_rates：被 visualize.py 用来绘制训练历史曲线。
  • config：保存模型结构和训练超参数，加载 checkpoint 时可复原模型尺寸。

Transformer 翻译任务是怎么工作的

Transformer 最早就是在机器翻译任务中提出的。相比单纯写一个 copy task，翻译任务更接近真实的 seq2seq 场景。

一个翻译模型可以粗略理解成：

英文句子 → Encoder → 语义表示 memory
中文前缀 → Decoder → 预测下一个中文 token

训练时，模型输入的是：

src        = 英文 token 序列
tgt_input  = <bos> + 中文 token 序列
tgt_output = 中文 token 序列 + <eos>

例如：

EN: I love you
ZH: 我爱你

经过处理后：

src        = [I, love, you, <eos>]
tgt_input  = [<bos>, 我, 爱, 你]
tgt_output = [我, 爱, 你, <eos>]

tgt_input 和 tgt_output 错位一位，这就是 Teacher Forcing。训练时 decoder 每一步看到前面的真实 token，学习预测下一个 token。

推理时则没有真实答案，只能自回归生成：

<bos>
<bos> 我
<bos> 我 爱
<bos> 我 爱 你
...

Tokenizer：把文本变成模型能读的数字

模型不能直接处理字符串，它只能处理数字。Tokenizer 的作用就是把文本转成 token id。

这里使用 SentencePiece 的 BPE。它和 GPT-2 的 BPE 词表本质上是同一类东西，只不过 GPT-2 使用的是自己的 byte-level BPE 词表，而这里是基于 AI Challenger 语料训练一个中英文共享词表。

Tokenizer 并不理解翻译关系。它不知道哪句英文对应哪句中文，它只是统计语料中哪些字符组合、子词组合经常出现，然后构建一个 subword 词表。

import sentencepiece as spm

sp = spm.SentencePieceProcessor()
sp.load("data/spm/bpe.model")

text = "I love machine learning."
ids = sp.encode(text, out_type=int)
pieces = sp.encode(text, out_type=str)

print(pieces)
print(ids)
print(sp.decode(ids))

可能输出：

['▁I', '▁love', '▁machine', '▁lear', 'ning', '.']
[26, 750, 2799, 2761, 423, 12888]
I love machine learning.

其中 ▁ 表示词开始，也就是原文本里的空格位置。

Dataset：构造 src、tgt_input、tgt_output

数据集这一层负责把中英文句子转换成训练需要的张量。

核心逻辑是：

src        = English tokens + <eos>
tgt_input  = <bos> + Chinese tokens
tgt_output = Chinese tokens + <eos>

由于每个句子的长度不同，一个 batch 里的样本必须 padding 到同一长度。

padding：让 tensor 能拼成 batch
mask   ：告诉模型哪些位置不能看

例如，一个 batch 中有两句话：

句子1: I love you
句子2: I love machine learning

假设已经 token 化为 id：

句子1: [5, 20, 33]
句子2: [5, 20, 102, 88]

因为长度不同（3 vs 4），需要 padding 到相同长度：

句子1: [5, 20, 33, 0]
句子2: [5, 20, 102, 88]

其中 0 是 <pad> 的 id。

此时 batch 可以拼成一个 tensor：

[
  [5, 20, 33, 0],
  [5, 20, 102, 88]
]

接下来需要 mask，告诉模型不要关注 padding：

padding mask:
[
  [1, 1, 1, 0],
  [1, 1, 1, 1]
]

其中：

1 表示可以参与计算
0 表示需要被屏蔽（padding 位置）

在 attention 计算时，这些为 0 的位置会被置为 -inf，从而在 softmax 后权重为 0。

当前项目没有把所有文本一次性读进内存。对于 AI Challenger 这种较大的语料，更合理的方式是先构建文件偏移量，然后按需读取样本。这样可以减少不必要的内存占用。

Mask：Padding Mask 和 Causal Mask

Padding mask 用来忽略 <pad>：

def create_padding_mask(seq, pad_id):
    # seq: [B, T]
    return (seq != pad_id).unsqueeze(1).unsqueeze(2)

返回形状是：

[B, 1, 1, T]

它可以 broadcast 到 attention score 的形状：

[B, H, Tq, Tk]

Causal mask 用在 decoder masked self-attention 中，防止模型看到未来 token：

def create_causal_mask(size):
    mask = torch.tril(torch.ones(size, size)).bool()
    return mask.unsqueeze(0).unsqueeze(0)

如果序列长度是 4，它长这样：

1 0 0 0
1 1 0 0
1 1 1 0
1 1 1 1

第 1 个 token 只能看自己，第 2 个 token 可以看第 1 个和自己，依此类推。

Multi-Head Attention

Attention 的核心公式是：

Attention(Q,K,V)=softmax(QKTdk)V\mathrm{Attention}(Q,K,V)=\mathrm{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)VAttention(Q,K,V)=softmax(dk​​QKT​)V

Self-Attention 中：

Q = K = V = x

Cross-Attention 中：

Q = decoder 当前状态
K = encoder 输出 memory
V = encoder 输出 memory

代码中的核心实现如下：

# src/model.py
import torch
import torch.nn as nn
import torch.nn.functional as F


class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads, dropout=0.1):
        super().__init__()
        assert d_model % num_heads == 0

        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads

        self.Wq = nn.Linear(d_model, d_model)
        self.Wk = nn.Linear(d_model, d_model)
        self.Wv = nn.Linear(d_model, d_model)
        self.out = nn.Linear(d_model, d_model)
        self.dropout = nn.Dropout(dropout)

    def forward(self, Q, K, V, mask=None):
        B, Tq, D = Q.size()
        _, Tk, _ = K.size()
        _, Tv, _ = V.size()

        Q = self.Wq(Q)
        K = self.Wk(K)
        V = self.Wv(V)

        Q = Q.view(B, Tq, self.num_heads, self.d_k).transpose(1, 2)
        K = K.view(B, Tk, self.num_heads, self.d_k).transpose(1, 2)
        V = V.view(B, Tv, self.num_heads, self.d_k).transpose(1, 2)

        scores = Q @ K.transpose(-2, -1) / (self.d_k ** 0.5)

        if mask is not None:
            scores = scores.masked_fill(mask == 0, float("-inf"))

        attn = F.softmax(scores, dim=-1)
        attn = self.dropout(attn)
        out = attn @ V

        out = out.transpose(1, 2).contiguous().view(B, Tq, D)
        out = self.out(out)
        return out

这里有一个容易写错的细节：Cross-Attention 时 Tq 和 Tk 不一定相等。Decoder 当前长度和 Encoder 输入长度可能不同，所以不能简单写成：

B, T, D = Q.size()

然后所有地方都用同一个 T。

Multi-Head Attention 中 Q/K/V reshape 过程

Encoder 和 Decoder 的区别

Encoder 负责读入完整源句子，输出一组上下文表示：

src tokens → encoder → memory

Decoder 负责根据已经生成的目标前缀，以及 encoder 的 memory，预测下一个 token：

tgt prefix + memory → decoder → next token logits

两者最大的区别在于 attention 结构：

Encoder Layer:
Self-Attention
Feed Forward

Decoder Layer:
Masked Self-Attention
Cross-Attention
Feed Forward

Decoder 多了两件事：

第一，Masked Self-Attention 不能看未来 token，否则训练时会直接偷看答案。

第二，Cross-Attention 要去看 Encoder 的输出 memory，这也是原论文结构图中从左边 Encoder 连到右边 Decoder 的那条线。

关于 KV Cache

这个 seq2seq demo 当前还没有实现 KV Cache，但理解推理阶段时，KV Cache 非常重要。

Decoder 每一步生成一个新 token。对于当前步来说，只需要当前 token 的 Query，但需要所有历史 token 的 Key 和 Value。

训练时会一次性计算完整 attention 矩阵：

QK^T

但推理时只关心最后一个 token 的输出，也就是只需要最后一行：

      K1  K2  K3  ...  Kt
Q1    ·   ·   ·        ·
Q2    ·   ·   ·        ·
...
Qt    ·   ·   ·        ·   ← 推理时只需要这一行

所以推理阶段可以缓存历史 K/V：

K_cache = [K1, K2, ..., Kt-1]
V_cache = [V1, V2, ..., Vt-1]

每来一个新 token，只计算：

Q_t, K_t, V_t

然后把 K_t 和 V_t 拼到 cache 中。Q 不需要缓存，因为当前步只使用当前 Q。
