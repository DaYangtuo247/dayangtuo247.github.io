CLER Improving Multimodal Financial Reasoning by Cross-MLLM Error Reflection

Apr 14, 2026
17 min read

P0. 基本信息

  • 题目： CLER: Improving Multimodal Financial Reasoning by Cross-MLLM Error Reflection (CLER：通过跨多模态大模型的错误反思提升金融推理能力)
  • 会议/期刊： AAAI-26 (The Fortieth AAAI Conference on Artificial Intelligence)
  • 年份： 2026年 (注：按论文版权页标识为2026，即被AAAI最新录用)
  • 作者： Shuangyan Deng, Zhongsheng Wang, Rui Mao, Ciprian Doru Giurcaneanu, Jiamou Liu
  • 单位： 新西兰奥克兰大学，新加坡南洋理工大学

P1. 问题定义

在量化金融和金融分析中，我们经常需要同时处理文本（如财报、新闻）和视觉信息（如K线图、财务表格）。这就引出了多模态金融推理任务。然而，目前的商业多模态大模型（MLLMs，如GPT-4o）面临两个致命痛点：

  1. 推理能力依然不足： 在面对复杂的金融术语、逻辑一致性和数值计算时，大模型的准确率往往不到60%，经常犯一些基础的公式误用或正负号看错的错误（参考原文图1的错误示例）。
  2. 算力成本过于高昂： 处理高分辨率金融图表会引发“Token爆炸”。如果在推理时让商业闭源模型进行大规模的试错和纠错，会产生无法承受的API成本和延迟。

P2. 前人工作

为了解决大模型容易犯错的问题，前人提出了“从错误中学习”的思想，主要分为两类途径，但都不太适用于我们的金融场景：

第一类：利用负面数据对大模型进行微调 (Fine-tuning with negative data)

  • 核心思想： 将大模型常犯的错误作为负样本喂给模型，重新训练，让它学会“不要这样做”。
  • 代表性方法： 传统的基于人类反馈的强化学习（RLHF）或负样本对齐微调。
  • 主要缺点： 极度不灵活且成本巨大。对于GPT-4o这种闭源模型，我们根本无法进行微调；对于开源多模态大模型，微调所需的数据标注成本和算力成本也极高。

第二类：基于外部错误数据库的通用原则生成 (General Error Retrieval & Principle Generation)

  • 核心思想： 将历史错误存入数据库，推理时根据当前问题检索出相关的错误案例，并总结出一段“通用指导原则”（如LEAP和RICP方法），拼接到提示词中。
  • 代表性方法： LEAP (2024), RICP (2024)。
  • 主要缺点：
    1. 粗粒度、无针对性： 多步金融推理（如计算现值）步骤繁多。前人方法生成的“通用原则”过于宽泛（比如仅仅提示“注意公式计算”），无法精准制导具体的计算步骤错误。
    2. 检索方式存在缺陷： 仅依赖表面文本相似度检索。如果两个问题看起来相似但底层金融逻辑不同，检索出的错误案例反而会误导模型（幻觉）。
    3. Token成本依然很高： 动态生成原则的过程需要大模型反复消耗多模态Token。

P3. 本文改进思路、创新点和贡献

为了克服上述缺陷，本文提出了一种具有高度成本效益的框架 CLER (Contrastive Learning for Error Retrieval)。

  • 核心思路： “智者从别人的错误中学习”。既然商业大模型API太贵，我们就让免费的开源大模型去做金融题，把它们犯的步骤错误收集起来建库。当商业模型做题时，如果在某一步不确定，就去开源数据库里检索结构和语义都最相似的错误步骤作为反面教材，引导商业模型进行个性化的自我反思。
  • 主要贡献：
    1. 提出了CLER框架，首次利用跨模型错误反思机制提升金融推理。
    2. 构建并开源了首个包含8000+条“错误-纠正”步骤对的金融多模态推理错误数据集 FinErrorSet。
    3. 训练了一个基于对比学习的细粒度检索器，克服了传统文本检索的表面性问题。

与前人工作的对比改进如下表：

P2中前人方法的缺陷本文CLER做出的改进
闭源模型无法微调采用即插即用的Prompt反思机制，无需改变模型参数。
高昂的多模态试错成本错误数据库由免费的开源模型（如Qwen-VL）离线生成，商业模型仅在推理测试阶段消耗成本。
通用原则过于宽泛、缺乏指导性细化到步骤级 (Step-level)的检索与纠错，引导模型针对自身独特的推理路径生成个性化反思 (Individualized reflection)。
传统文本检索容易引入误导性案例引入对比学习 (Contrastive Learning) 训练检索器，不仅看字面意思，更捕捉金融推理的逻辑结构相似性。

P4. 本文研究使用的重要公式

本论文的理论基础非常清晰，主要通过以下几个核心公式来定义问题和检索模型：

1. 多模态金融推理任务定义（原文 Page 3）： 给定输入实例：X=(c,q,I,O)X = (c, q, I, O)X=(c,q,I,O)

  • ccc: 文本上下文（Context）
  • qqq: 推理问题（Question）
  • I={i1,…,is}I = \{i_1, \dots, i_s\}I={i1​,…,is​}: 视觉输入（如财报图表）
  • O={o1,…,on}O = \{o_1, \dots, o_n\}O={o1​,…,on​}: 候选项 目标是让模型生成一个多步推理轨迹 R^={r^1,…,r^T}\hat{\mathcal{R}} = \{\hat{r}_1, \dots, \hat{r}_T\}R^={r^1​,…,r^T​}，最终导向正确答案 A∗A^*A∗。

2. 错误集定义（原文 Page 4）： 我们构建的数据库 D\mathcal{D}D 包含了开源模型犯错的“错误-纠正”步骤对：

D={(r1,1′,r1,1∗),…,(ri,j′,ri,j∗)}\mathcal{D} = \{(r'_{1,1}, r^*_{1,1}), \dots, (r'_{i,j}, r^*_{i,j})\}D={(r1,1′​,r1,1∗​),…,(ri,j′​,ri,j∗​)}

其中 r′r'r′ 代表错误步骤，r∗r^*r∗ 代表该步骤的正确修正版。

3. 对比学习检索器的损失函数（原文 Page 5, Algorithm 1）： 为了让检索器能找到逻辑结构相似的错误，作者使用了改进的InfoNCE损失函数：

Li=−log⁡exp⁡(simi+/τ)∑sim∈simi−exp⁡(sim/τ)\mathcal{L}_i = -\log \frac{\exp(\text{sim}_i^+/\tau)}{\sum_{\text{sim} \in \text{sim}_i^-} \exp(\text{sim}/\tau)}Li​=−log∑sim∈simi−​​exp(sim/τ)exp(simi+​/τ)​

  • 这里，锚点 (Anchor) 是我们当前的推理步骤编码 eie_iei​。
  • 正样本 (Positive, simi+\text{sim}_i^+simi+​) 是对应的修正步骤 ei+e^+_iei+​ 的余弦相似度。
  • 负样本 (Negative, simi−\text{sim}_i^-simi−​) 是来自不同问题或不同结构的错误步骤。
  • 通过最小化该损失，模型被迫将“表述不同但逻辑错误一致”的步骤拉近，将“表面相似但逻辑不同”的步骤推开。

P5. 技术框图

  1. 模块(a) - 错误数据库构建 (FinErrorSet)： 利用三个开源的多模态大模型（DeepSeek-VL, InternVL, Qwen-VL）去解答复杂的金融问题，强制它们使用链式思考（CoT），并使用充当“Step Checker”的模型对错误步骤进行识别和纠正，形成一个包含“错题和正确解答”的知识库。
  2. 模块(b) - 检索器对比学习训练： 使用刚才讲过的对比学习公式，训练一个专门的检索器（Retriever）。它的目标是在海量的错题本中，精准定位到与当前推理卡壳处最相关的历史错误。
  3. 模块(c) - 步骤级反思与纠正 (Reflection and correction)： 在真实应用场景中，商业大模型（比如你花钱调用的GPT-4o）在生成每一步推理时，会先通过检索器去数据库里“看一眼别人犯过的类似错误”，生成一段自己的反思，然后修正当前步骤，继续往下推理。

P6. 框图中核心步骤详解

为了让你更清楚CLER在工程上是如何实现的，我们进一步拆解核心的第三步“反思与纠正”的实际工作流：

  • 前向推理： 商业大模型正在解答一道求债券现值的问题，它生成了初步的推理链 R={r1,r2,…,rT}\mathcal{R} = \{r_1, r_2, \dots, r_T\}R={r1​,r2​,…,rT​}。
  • 细粒度拦截： 对于当中的某一步 rtr_trt​，系统立刻将其作为Query，输入给训练好的对比学习检索器。
  • Top-1 检索： 检索器从 FinErrorSet 数据库中找出结构最匹配的开源模型错误步骤 ri,j′r'_{i,j}ri,j′​ 以及对应的正确修正 ri,j∗r^*_{i,j}ri,j∗​。（你可以回看原文图1的那个红色的打叉步骤和绿色修正步骤）。
  • 生成个性化见解 (Insight Generation)： 大模型通过对比自己当前的 rtr_trt​ 和检索出的 (ri,j′,ri,j∗)(r'_{i,j}, r^*_{i,j})(ri,j′​,ri,j∗​)，执行一个反思函数：I(rt,ri,j′,ri,j∗)\mathcal{I}(r_t, r'_{i,j}, r^*_{i,j})I(rt​,ri,j′​,ri,j∗​)。模型会说：“哦，我明白了，刚才那个开源模型在这里忘了折现率是针对两年的，我必须注意不要犯同样的错。”
  • 自我纠正 (Self-correction)： 最终输出修正后的可靠步骤 r^t=Self-correction(rt,I)\hat{r}_t = \text{Self-correction}(r_t, \mathcal{I})r^t​=Self-correction(rt​,I)。

这种步骤级的防御，就像是给商业模型配备了一个实时的“代码审查员”。

P7. 实验数据、指标和实现细节

为了验证CLER的有效性，作者进行了详尽的实验：

  • 测试数据集： 选取了三个公认的、极具挑战性的多模态金融推理基准：
    1. FinMR: 专家级金融推理，包含复杂长文本和图表。
    2. FAMMA: 强调跨语言能力和大学水平的金融图表推理。
    3. MMMU-Finance: 包含经济、会计、金融等重度图表（95%是表格）的数据集。
  • 评估指标： 选择题的准确率 (Accuracy %)。
  • 对比基线模型 (Baselines)：
    • CoT (思维链): 直接让模型自己一步步想。
    • LEAP / RICP: 目前SOTA的基于错误原则学习的通用方法。
  • 实现细节： 错误库使用 Qwen2.5-VL-72B, DeepSeek-VL-2 等开源模型生成；被测的主模型则是闭源商业巨头：GPT-4o, Gemini-2.0-flash, Claude-4-sonnet。温度参数设为0.7。

P8. 实验结果一

结论非常清晰：CLER在所有模型和所有数据集上全面超越了基线方法。

  • 绝对提升显著： 在要求最高专业度的FinMR (Expertise) 子集上，CLER使得GPT-4o的准确率达到了 65.90%，比单纯的思维链（CoT）高出了惊人的 8.83%！
  • 为什么击败了LEAP和RICP？ 表格中我们注意到一个有趣的现象（带有下划线的数据），LEAP等基于通用原则的方法，在处理涉及密集计算的Math子集时，准确率有时甚至会下降（比如在FinMR的Math子集上下降了6.43%）。这是因为金融计算错误对结构非常敏感，传统的文本相似度检索找错了“原则”，导致大模型被误导。而CLER因为采用了结构化的对比学习检索和步骤级纠错，不仅没有下降，反而稳定提升（+4.01%）。

P9. 实验结果二

  1. 消融实验（Table 3）：模块到底有没有用？
    • 如果去掉“反思（Reflection）”，直接让模型看答案，准确率会大幅掉落。
    • 如果将“对比学习检索器（CL Retriever）”退化成普通的语义检索，效果也会明显下降。这证明了**“结构化检索+个性化反思”**是缺一不可的黄金组合。

  1. 个性化指导的作用（Figure 3）： 图3对比了“使用预先生成的通用原则”和“让大模型动态反思生成个性化见解”。可以看到，针对自己独特的推理路径进行的反思（绿色柱子），比死记硬背通用原则（蓝色柱子）要高出3%左右的准确率。

  2. 错误库规模的影响（Figure 4）： 图4展示了随着 FinErrorSet 数据库大小的改变（100% -> 80% -> 50%），模型准确率呈线性下降。这说明量化工程中常说的“数据飞轮”在这里也适用——开源模型踩的坑越多，数据库越大，商业模型能避开的雷也就越多。

P10. 总结

总结一下，这篇论文为解决大模型在金融领域水土不服的问题提供了一个极佳的工程思路。

核心亮点：

  1. 提出了一个高度成本可控的架构，利用低成本的开源模型白嫖错误经验，去喂养高成本的商业模型。
  2. 实现了步骤级别的纠错，极大地提升了复杂的数值计算和长逻辑推理准确率。
  3. 构建了有价值的 FinErrorSet 数据集和对比学习检索器。

论文局限性（原文坦诚提及）：

  1. 系统的天花板取决于错误数据库的多样性，如果出现一种所有开源模型都没犯过的新型错误，检索器就束手无策了。
  2. 目前只关注“单一连贯步骤”的纠错，对于跨越长周期的、高阶的多步连锁逻辑错误，捕捉能力仍需加强。
