Transformer系列 - 位置编码

Transformer 本身没有像 RNN 那样“一个词一个词按顺序传下去”的结构。 所以如果只把词向量喂进去：

  • “猫 追 鼠”
  • “鼠 追 猫”

模型看到的都只是这几个词的 embedding，它不知道谁在前、谁在后。

所以要额外加一个东西，告诉模型：

  • 这是第 0 个词
  • 这是第 1 个词
  • 这是第 2 个词

这个东西就是 Position Encoding（位置编码）。

二、例子

假设我们有一句话：

[猫, 追, 鼠]

位置从 0 开始编号：

  • “猫” 在 pos = 0
  • “追” 在 pos = 1
  • “鼠” 在 pos = 2

为了方便计算，假设模型维度：

dmodel=4d_{model} = 4dmodel​=4

也就是说，每个位置最后会得到一个 4 维向量。

三、位置编码公式

对每个位置 pos，它的 4 维编码向量 ( PE(pos) ) 的每个维度 ( i )（( i=0,1,2,3 )）计算方式如下：

当 ( i ) 是偶数（( i=0,2 )）时，用正弦函数：

PE(pos,i)=sin⁡(pos10000(2i/dmodel))PE(pos, i)=\sin\left(\frac{pos}{10000^{(2i/d_{model})}}\right)PE(pos,i)=sin(10000(2i/dmodel​)pos​)

当 ( i ) 是奇数（( i=1,3 )）时，用余弦函数：

PE(pos,i)=cos⁡(pos10000(2i/dmodel))PE(pos, i)=\cos\left(\frac{pos}{10000^{(2i/d_{model})}}\right)PE(pos,i)=cos(10000(2i/dmodel​)pos​)

这里的

10000(2i/dmodel)10000^{(2i/d_{model})}10000(2i/dmodel​)

可以理解成一个“缩放因子”，它控制不同维度变化得快还是慢。

你可以先不纠结为什么是 10000，先把它看成一个固定设定。

四、先算“缩放因子”

因为这里 dmodel=4d_{model}=4dmodel​=4

所以： 2idmodel=2i4=i2\frac{2i}{d_{model}}=\frac{2i}{4}=\frac{i}{2}dmodel​2i​=42i​=2i​

于是缩放因子就变成： 10000i/210000^{i/2}10000i/2

分别算 4 个维度：

  • i = 0： 100000/2=100000=110000^{0/2}=10000^0=1100000/2=100000=1

  • i = 1： 100001/2=10000=10010000^{1/2}=\sqrt{10000}=100100001/2=10000​=100

  • i = 2： 100002/2=100001=1000010000^{2/2}=10000^1=10000100002/2=100001=10000

  • i = 3： 100003/2=10000×10000=10000×100=100000010000^{3/2}=10000\times \sqrt{10000}=10000\times 100=1000000100003/2=10000×10000​=10000×100=1000000

五、手动计算编码

现在我们来算第一个词“猫”，位置是：pos=0pos=0pos=0，逐个维度代入。

  • 第 0 维：i = 0（偶数，用 sin）： PE(0,0)=sin⁡(01)=sin⁡(0)=0PE(0,0)=\sin\left(\frac{0}{1}\right)=\sin(0)=0PE(0,0)=sin(10​)=sin(0)=0

  • 第 1 维：i = 1（奇数，用 cos）： PE(0,1)=cos⁡(0100)=cos⁡(0)=1PE(0,1)=\cos\left(\frac{0}{100}\right)=\cos(0)=1PE(0,1)=cos(1000​)=cos(0)=1

  • 第 2 维：i = 2（偶数，用 sin）： PE(0,2)=sin⁡(010000)=sin⁡(0)=0PE(0,2)=\sin\left(\frac{0}{10000}\right)=\sin(0)=0PE(0,2)=sin(100000​)=sin(0)=0

  • 第 3 维：i = 3（奇数，用 cos）： PE(0,3)=cos⁡(01000000)=cos⁡(0)=1PE(0,3)=\cos\left(\frac{0}{1000000}\right)=\cos(0)=1PE(0,3)=cos(10000000​)=cos(0)=1

所以，“猫”的位置编码是：

PE(0)=[0,1,0,1]PE(0)=[0,1,0,1]PE(0)=[0,1,0,1]

以此类推三个词的位置编码如下

猫  pos=0 -> [0, 1, 0, 1]
追  pos=1 -> [0.841, 0.9999, 0.0001, 1.0]
鼠  pos=2 -> [0.909, 0.9998, 0.0002, 1.0]

可以看到：

  • 每个位置的编码都不一样
  • 所以模型可以靠这些向量区分“谁在第几个位置”

六、这些编码到底有什么用？

1. 每个位置的编码都不同

比如：

  • “猫” 的编码是 ([0,1,0,1])
  • “追” 的编码是 ([0.841,0.9999,0.0001,1.0])
  • “鼠” 的编码是 ([0.909,0.9998,0.0002,1.0])

它们完全不同，所以模型可以区分：

  • 哪个词在前
  • 哪个词在后
  • 谁是第 0 个位置
  • 谁是第 1 个位置

2. 位置变化是有规律的

用一句话总结就是：相差 1 个位置，变化方式有规律

比如从 pos=1 到 pos=2：

  • 第 0 维：0.841→0.9090.841 \to 0.9090.841→0.909
  • 第 1 维：0.9999→0.99980.9999 \to 0.99980.9999→0.9998
  • 第 2 维：0.0001→0.00020.0001 \to 0.00020.0001→0.0002

你会发现：

  • 有的维度变化快
  • 有的维度变化慢
  • 但整体变化是由 sin/cos 决定的，不是乱跳的

所以它不是普通的“编号标签”，而是一种有结构的编码。

七、为什么不能直接用序号？

这是最关键的理解部分。 你可能会想：

pos=0,1,2,3...

不就行了吗？直接把编号告诉模型，多简单。 这个想法表面上很自然，但有几个问题。

1. 直接用序号，只有“绝对位置”，没有好的结构

如果直接用：

猫 -> 0
追 -> 1
鼠 -> 2

模型确实知道谁在第几个位置。

但问题是：

  • 1 和 2 的差是 1
  • 100 和 101 的差也是 1

表面上都差 1， 但对神经网络来说，这两个数值区间差太大了，不容易学出统一规律。

也就是说，模型不容易自然学会：

“不管当前位置是 1 还是 100，只要加 1，就表示往后挪一个位置”

2. 直接用序号，尺度会越来越大

句子长一点时，位置可能变成：

0,1,2,...,1000,2000...

数值会不断增大。

这样会带来两个问题：

  • 数值范围越来越大
  • 不同长度句子的分布不稳定

而 sin/cos 编码始终落在：[−1,1][-1,1][−1,1] 范围内，更稳定。

3. 直接用序号，不适合表达“相对位置”

NLP 里很多时候更重要的不是“它是第 17 个词”，而是：

  • 它和前一个词隔多远
  • 它和主语隔多远
  • 它和谓语隔多远

也就是：相对位置 比 绝对编号 更重要

sin/cos 编码的好处在于，它把位置变成一种带规律的向量表示，更容易让模型通过向量关系去感知距离。

八、位置编码真正厉害的地方

它厉害的地方不是：

差值固定

而是：

位置变化是由一套统一公式生成的

也就是：

  • pos=0 有自己的编码
  • pos=1 有自己的编码
  • pos=2 有自己的编码
  • 所有位置都是同一种规则生成出来的

所以模型面对不同长度、不同位置时，看到的是同一种结构化信号，而不是简单的编号。

九、一个更直觉的理解

你可以把它想成：直接编号

像这样：

0,1,2,3,4,5...

只是数字标签。

sin/cos 位置编码

像这样：

每个位置都对应一组波形坐标

不同维度是不同频率的波：

  • 有的变化快
  • 有的变化慢

所以每个位置都会对应一个独特的“波形组合”。

这样模型不只是知道：它是第几个

还更容易学到：它和别的位置之间是什么关系

十、和词向量怎么结合？

Transformer 里通常不是单独用位置编码，而是：

输入向量=词向量+位置编码输入向量 = 词向量 + 位置编码输入向量=词向量+位置编码

也就是：

x=embedding+PEx = embedding + PEx=embedding+PE

这样一个词最终输入给模型时，同时带了两部分信息：

  • 这个词是什么
  • 这个词在什么位置

比如：

  • “猫”的 embedding 负责表示“猫”的语义
  • “猫”的 position encoding 负责表示“猫”在句子中的位置

两者加起来，模型才知道：这是“猫”，并且它在前面
