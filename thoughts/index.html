2026.07.06

阅读了 Claude Code Blog 关于 HTML 比 Markdown 的优势 的文章，觉得很有道理。

核心观点：

  • Markdown 的语法有限，很多时候需要自己去扩展
  • HTML 本身拥有丰富的标签库，可以表达更复杂的内容
  • HTML 的语法比 Markdown 更灵活，支持更多的样式和功能

实践感受： 今天在改造 Blog，开始尝试使用 HTML + Markdown 混合的方式来写文章。这种混合方式相比纯 Markdown，确实要舒服很多——既能享受 Markdown 的简洁，又能用 HTML 处理更复杂的排版需求。 example: HTML 帖子：比 Markdown 表达更多
